説明

第2回ではサーバを作り、ブラウザにHTMLやJSONを返しました。
今回はクライアント（ブラウザ）のJavaScriptからサーバにデータを送受信する方法を学びます。これにより、ページを再読み込みせずにデータを取得・送信できるようになります。

1.1 JSON とは 課題に出ます

友達と連絡先を交換するとき、「名前: 田中、電話: 090-xxxx、趣味: 読書」のように項目名と値のペアで書きます。JSON はコンピュータ同士がデータを交換するときの「共通フォーマット」で、同じようにキーと値のペアで構造化されたデータです。

JSON（JavaScript Object Notation）は、データを構造化して表現するテキスト形式です。Webアプリケーションでのデータ通信で最も広く使われています。

JSON
{
  "name": "田中太郎",
  "age": 20,
  "hobbies": ["読書", "プログラミング", "カフェ巡り"],
  "university": {
    "name": "○○大学",
    "year": 2
  }
}
JSONの要素説明例
キーデータの項目名。必ずダブルクォートで囲む"name", "age"
文字列テキストデータ。ダブルクォートで囲む"田中太郎"
数値数字。クォートは不要20
配列複数の値をまとめる。角括弧 [] で囲む["読書", "プログラミング"]
オブジェクト入れ子のキー・値ペア。波括弧 {} で囲む{"name": "○○大学"}
真偽値true または falsetrue
null値がないことを明示するnull

1.2 Fetch API ── ブラウザからサーバにリクエストを送る 課題に出ます

普段ブラウザでURLを入力してページを見るとき、ブラウザが裏で自動的にリクエストを送っています。Fetch API を使うと、JavaScriptのコードから手動でリクエストを送れるようになります。電話（URLバー）ではなく、プログラムでメールを送るようなものです。

Fetch API はブラウザに標準搭載されている機能で、JavaScriptからHTTPリクエストを送信し、サーバからのレスポンスを受け取ることができます。

JavaScript
// サーバの /api/data にGETリクエストを送る
fetch('/api/data')
  .then(function(response) {
    return response.json();  // レスポンスをJSONとして解析
  })
  .then(function(data) {
    console.log(data);      // 受け取ったデータを使う
  });

1.3 async/await 基礎 課題に出ます

レストランで料理を注文すると、料理ができるまで待つ必要があります。await はまさにこの「待つ」を表すキーワードです。サーバからデータが届くまで待ち、届いたら次の処理を進めます。

fetch() はサーバとの通信を行うため、結果がすぐには返ってきません（非同期処理）。async/await を使うと、非同期処理を同期的（上から順番に実行されるよう）に書けます。

JavaScript
// async 関数の中でのみ await が使える
async function loadData() {
  const response = await fetch('/api/data');   // サーバにリクエスト → 返答を待つ
  const data = await response.json();          // JSONを解析 → 完了を待つ
  console.log(data);                            // データを使う
}
キーワード意味
async関数を非同期関数として宣言する。この関数の中で await が使えるようになる
await非同期処理の完了を待つ。結果が返ってくるまで次の行に進まない

await を使わないと、サーバの返答が届く前に次の処理が進んでしまい、data が undefined（未定義）になります。

1.4 GET と POST の使い分け 課題に出ます

GET は図書館で「この本を見せてください」と頼むこと（情報を取得する）。POST は窓口に「この申込書を提出します」と書類を渡すこと（情報を送信する）。
GETPOST
目的データを取得するデータを送信・作成する
データの場所URLに含まれる（クエリ文字列）リクエストの本文（body）に含まれる
ブラウザ操作URLバーにアドレスを入力する = GETフォームの送信ボタン、JavaScript の fetch
例記事一覧の取得、プロフィール表示コメント投稿、ログインフォーム送信
Expressapp.get()app.post()
JavaScript
// サーバ側: POST リクエストを受け取るルート
app.post('/api/messages', function(req, res) {
  const body = req.body;  // 送信されたデータを受け取る
  res.json({ status: 'received', data: body });
});

POST で送られたデータを受け取るには 課題に出ます

Express でPOSTリクエストのbodyを受け取るには、ミドルウェアを追加する必要があります: app.use(express.json()) ── これにより req.body でJSONデータにアクセスできるようになります。

1.5 REST API の考え方

図書館の本は「ジャンル → 著者名 → タイトル」のように整理されています。REST API も同じで、データをリソース（資源）として整理し、URLで「どのデータにアクセスするか」を表現します。

REST（Representational State Transfer）は、WebAPIの設計思想です。URLでリソースを表し、HTTPメソッドで操作を表します。

HTTPメソッドURL意味
GET/api/usersユーザ一覧を取得
GET/api/users/1ID=1 のユーザを取得
POST/api/users新しいユーザを作成
PUT/api/users/1ID=1 のユーザを更新
DELETE/api/users/1ID=1 のユーザを削除

この授業では GET と POST を主に使います。URLは「何のデータか」を表し、メソッドは「何をするか」を表すのがポイントです。

1.6 DOM操作でデータを表示する 課題に出ます

ホワイトボードに書かれた内容を消して新しく書き直すように、JavaScriptでHTMLの中身を動的に書き換えることができます。これがDOM操作です。

サーバから取得したデータをページに表示するには、DOM（Document Object Model）を操作してHTMLの内容を更新します。

JavaScript
// HTML: <div id="result"></div>

// JavaScriptでこの要素を取得し、中身を書き換える
const resultDiv = document.getElementById('result');
resultDiv.innerHTML = '<p>サーバから取得したデータです</p>';
メソッド / プロパティ説明
document.getElementById('id')指定したIDを持つHTML要素を取得する
要素.innerHTML要素の中身のHTMLを読み書きする
要素.textContent要素のテキスト内容を読み書きする（HTMLタグは反映しない）

Fetch API でデータを取得 → DOM操作で表示。この2つを組み合わせることで、ページを再読み込みせずにサーバのデータを画面に反映できます。

  <script>
    async function sendMessage() {
      const name = document.getElementById('nameInput').value;
      const msg  = document.getElementById('msgInput').value;

      const response = await fetch('/api/message', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ name: name, message: msg })
      });

      const data = await response.json();
      document.getElementById('response').innerHTML =
        '<p>サーバ応答: ' + data.reply + '</p>' +
        '<p>受信時刻: ' + data.receivedAt + '</p>';
    }
  </script>
</body>

Step 2: サーバにPOSTルートを追加する

  1. VS Code 左側のエクスプローラ（Web03 フォルダ直下）で 「新しいファイル」アイコン → server2.js を作成

JavaScript ── server2.js
const express = require('express');
const app = express();

// JSONのPOSTデータを受け取るためのミドルウェア
app.use(express.json());

// 静的ファイル配信
app.use(express.static('public'));

// GET: メンバー一覧API（コード体験1と同じ）
app.get('/api/members', function(req, res) {
  res.json({
    team: 'Webプログラミングチーム',
    members: [
      { name: '田中', role: 'リーダー' },
      { name: '佐藤', role: 'デザイン' },
      { name: '鈴木', role: 'サーバ担当' },
      { name: '高橋', role: 'フロント担当' }
    ]
  });
});

// POST: メッセージ受信API
app.post('/api/message', function(req, res) {
  console.log('受信:', req.body);
  res.json({
    reply: req.body.name + 'さん、メッセージ「' + req.body.message + '」を受け取りました！',
    receivedAt: new Date().toLocaleString('ja-JP')
  });
});

app.listen(3000, function() {
  console.log('サーバ起動: http://localhost:3000');
});
