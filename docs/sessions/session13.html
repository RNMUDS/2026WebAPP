説明

今回から最終制作に入ります。
第13回は設計、第14回は実装・調整、第15回は発表です。
まずは「何を作るか」「どう作るか」を計画しましょう。

1.1 最終制作の要件 課題に出ます

家を建てる前に設計図を描くように、アプリを作る前に何を作るかを明確にしておくことが重要です。
設計なしに作り始めると、途中で方向がわからなくなり、完成しないリスクが高まります。

最終制作では、これまで学んだ技術を組み合わせてマルチユーザ3D Webアプリケーションを作ります。

使用技術（必須）

技術用途学んだ回
A-Frame 1.7.13D空間の構築第4〜8回
Node.js + Expressサーバの構築第2〜3回
Socket.IOリアルタイム通信第9〜12回

基本機能（必須） 設計課題に含む

#機能説明
1マルチプレイ（アバター同期）複数のブラウザからアクセスし、各ユーザのアバターが3D空間内に表示される。位置・回転がリアルタイムに同期される。
2吹き出しチャット3D空間内でテキストメッセージを送信でき、アバターの上に吹き出しとして表示される。

自由要素（個性を出す部分）

  • テーマ ── 空間のコンセプト（教室、宇宙、水中、美術館、ゲーム空間など）
  • 3Dデザイン ── オブジェクトの配置、色、テクスチャ、ライティング
  • 追加機能 ── オブジェクト操作、エフェクト、音声、スコア、DB永続化など

1.2 設計の進め方

以下の5ステップで設計を進めます。

1. テーマ決定 → 2. 機能一覧 → 3. 画面設計 → 4. 技術選定 → 5. ファイル構成
ステップ内容ポイント
1. テーマ決定「どんな3D空間を作るか」を決める自分が興味を持てるテーマを選ぶ。作りたいもののイメージを言葉にする。
2. 機能一覧必須機能 + 追加機能をリスト化する必須の2機能を先に書き、余力で追加する機能を優先順位付けする。
3. 画面設計3D空間のレイアウトをスケッチする手描きでOK。どこに何を配置するか、UIはどこに置くかを決める。
4. 技術選定各機能をどの技術で実現するか整理する過去の回で学んだ技術を振り返り、該当する技術を選ぶ。
5. ファイル構成プロジェクトのフォルダ・ファイル構成を決めるserver.js / index.html / style.css など、必要なファイルを洗い出す。

1.4 設計ドキュメントの例

以下は「バーチャル水族館」をテーマにした設計例です。自分の設計の参考にしてください。

設計例: バーチャル水族館

テーマ: 海の中を自由に泳げるバーチャル水族館。複数人が同時に入り、魚や海の生物を観察しながらチャットできる。

機能一覧:

区分機能使用技術
必須マルチプレイ（ダイバーアバター同期）Socket.IO + A-Frame
必須吹き出しチャット（泡のデザイン）Socket.IO + A-Frame text
追加魚の群れアニメーションA-Frame animation
追加チャット履歴の永続化SQLite + better-sqlite3
余力BGM（海の音）HTML5 Audio

ファイル構成:

ファイル構成
Web13/
  server.js          ← Express + Socket.IO サーバ
  index.html         ← A-Frame 3Dシーン + チャットUI
  style.css          ← チャットUIのスタイル
  package.json       ← npm 設定
  aquarium.db        ← SQLite データベース（自動生成）
  assets/
    fish.glb         ← 魚の3Dモデル
    coral.glb        ← サンゴの3Dモデル
    ocean-sound.mp3  ← BGM
