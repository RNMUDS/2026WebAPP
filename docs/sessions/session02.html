1.1 Node.js とは 課題に出ます

JavaScriptは本来ブラウザの中でしか動かない言語でした。Node.js は「ブラウザの外でもJavaScriptを動かせる実行環境」です。つまり、料理人（JavaScript）が厨房（ブラウザ）の外に出て、出張料理もできるようになったようなものです。

Node.js は、Google Chrome に搭載されている JavaScript エンジン「V8」を取り出して、サーバやPC上でJavaScriptを実行できるようにしたランタイム環境です。

特徴説明
V8エンジンGoogle Chrome と同じ高速なJavaScriptエンジンで動く
サーバ構築Webサーバを自分のプログラムで作れる。第1回で使ったLive Serverの代わりに自分でサーバを書く
同一言語ブラウザ側もサーバ側も JavaScript で書けるため、1つの言語でWebアプリ全体を構築できる
npm世界最大のパッケージ（ライブラリ）管理システムが付属。必要な機能を簡単に追加できる

Node.js をインストールすると、ターミナルで node ファイル名.js と打つだけでJavaScriptプログラムを実行できます。

1.2 npm とパッケージ管理 課題に出ます

料理をするとき、すべての調味料を自分で作る人はいません。醤油や味噌は買ってきます。プログラミングでも同じで、よく使う機能はパッケージ（ライブラリ）として公開されており、npm というツールで簡単にインストールできます。

npm（Node Package Manager）は Node.js に標準で付属するパッケージ管理ツールです。

コマンド何をするか
npm init -yプロジェクトを初期化し、package.json（設定ファイル）を自動生成する 課題に出ます
npm install expressExpress というパッケージをインストールする。node_modules/ フォルダに保存される
package.jsonプロジェクトの名前、バージョン、使用パッケージの一覧が記録されるファイル
node_modules/インストールしたパッケージの実体が入るフォルダ。手動で編集しない

1.3 Express.js フレームワーク 課題に出ます

Node.js だけでもWebサーバは作れますが、一から全部書くのは大変です。Express.js は「Webサーバを簡単に作るための骨組み（フレームワーク）」です。家を建てるとき、柱や壁の骨組みがあれば作業が楽になるのと同じです。

Express.js（通称 Express）は Node.js で最も広く使われるWebフレームワークです。HTTPリクエストの受信、ルーティング、レスポンスの送信を簡潔なコードで実現できます。

概念説明
appExpressアプリケーション本体。express() で生成する
ルーティング「どのURLに」「どのHTTPメソッドで」アクセスされたら「何を返すか」を定義すること 課題に出ます
ミドルウェアリクエストとレスポンスの間で実行される処理。ログ出力や静的ファイル配信などに使う

1.4 ルーティングの仕組み 課題に出ます

ホテルのフロントで「301号室に行きたい」と言うと案内されます。Webサーバも同じで、ブラウザが「/about にアクセスしたい」とリクエストすると、サーバが対応するルートを探して処理を実行します。

ルーティングとは、URLパスとHTTPメソッドの組み合わせに応じて、実行する処理を振り分ける仕組みです。

JavaScript
// GET メソッドで「/」にアクセスされたとき
app.get('/', function(req, res) {
  res.send('トップページです');
});
要素説明
app.get(パス, 関数)GETリクエストに対するルートを定義する
reqリクエストオブジェクト。ブラウザから送られた情報（URL、ヘッダなど）が入っている
resレスポンスオブジェクト。ブラウザに返す内容を操作するメソッドを持つ
パスマッチング'/' はトップ、'/about' は /about、'/api/hello' は /api/hello に対応

1.5 res.send() と res.json() の違い 課題に出ます

手紙を送るとき、「普通の文章」と「決まった書式の書類」は別物です。res.send() は自由な文章を返し、res.json() は JSON という決まったデータ形式で返します。
メソッド返すもの用途コード例
res.send()文字列やHTMLをそのまま返すWebページ、テキストメッセージres.send('<h1>Hello</h1>')
res.json()JavaScriptオブジェクトをJSON形式に変換して返すAPIレスポンス、データ通信res.json({ name: 'Taro' })

res.json() は自動的に Content-Type: application/json ヘッダを付けるため、クライアント側でデータとして処理しやすくなります。第3回のFetch APIで重要になります。

1.6 静的ファイル配信（express.static） 課題に出ます

レストランで「メニュー表」を見せるのに、毎回シェフが料理を作る必要はありません。あらかじめ印刷したメニューをそのまま渡すだけです。静的ファイル配信も同じで、HTML・CSS・画像などのファイルをそのままブラウザに返します。

Express では express.static() ミドルウェアを使うと、指定したフォルダ内のファイルを自動的にブラウザに配信できます。

JavaScript
// publicフォルダの中身をそのまま配信する
app.use(express.static('public'));

この1行で、public/index.html は http://localhost:3000/index.html でアクセスできるようになります。第1回の Live Server がやっていたこととほぼ同じ仕組みです。

ファイルパス（サーバ側）ブラウザでのURL
public/index.htmlhttp://localhost:3000/index.html
public/style.csshttp://localhost:3000/style.css
public/images/logo.pnghttp://localhost:3000/images/logo.png

コード体験 1

基本のExpressサーバ ── server.js

3つのルートを持つサーバを作ります。

  1. VS Code 左側のエクスプローラで 「新しいファイル」アイコン（紙に＋マーク）をクリック
  2. ファイル名を server.js と入力して Enter
  3. 下のコードブロック右上の「コピー」ボタンをクリックし、開いたファイルに 貼り付け（Ctrl+V / Cmd+V）
  4. 保存（Ctrl+S / Cmd+S）
JavaScript ── server.js
const express = require('express');
const app = express();

// ルート1: トップページ
app.get('/', function(req, res) {
  res.send('<h1>Hello Express!</h1><p>サーバが動いています</p>');
});

// ルート2: 自己紹介ページ
app.get('/about', function(req, res) {
  res.send('<h1>About</h1><p>これはWebプログラミング演習のサーバです</p>');
});

// ルート3: APIエンドポイント（JSON）
app.get('/api/hello', function(req, res) {
  res.json({ message: 'Hello from API!', timestamp: new Date().toISOString() });
});

// サーバ起動（ポート3000で待ち受け）
app.listen(3000, function() {
  console.log('サーバ起動: http://localhost:3000');
});

実行手順:

  1. VS Code のターミナルで以下を実行:

node server.js
  2. 「サーバ起動: http://localhost:3000」と表示されたら成功
  3. ブラウザで http://localhost:3000 を開く
  4. 次に http://localhost:3000/about を開く
  5. 次に http://localhost:3000/api/hello を開く

次に、サーバファイルを作ります。

  1. VS Code 左側のエクスプローラ（Web02 フォルダ直下）で 「新しいファイル」アイコン をクリック
  2. ファイル名を server2.js と入力して Enter
  3. 下のコードをコピーして貼り付け、保存する
JavaScript ── server2.js
const express = require('express');
const app = express();

// publicフォルダの中身を静的ファイルとして配信
app.use(express.static('public'));

// APIルート（静的ファイルとは別に動的レスポンスも返せる）
app.get('/api/info', function(req, res) {
  res.json({
    server: 'Express',
    staticDir: 'public',
    message: '静的ファイルとAPIを同時に提供しています'
  });
});

app.listen(3000, function() {
  console.log('サーバ起動: http://localhost:3000');
});

実行手順:

  1. もし前のサーバがまだ動いていたら Ctrl+C で停止する
  2. ターミナルで以下を実行: 
    Terminal
node server2.js
  3. ブラウザで http://localhost:3000 を開く → HTMLページが表示される
  4. 次に http://localhost:3000/api/info を開く → JSONデータが表示される
  5. 確認が終わったら Ctrl+C でサーバを停止

体験 コード体験 1 解答（記録例）
  • Q1: / → 「Hello Express!」の見出しと「サーバが動いています」の文。/about → 「About」の見出しと説明文。/api/hello → JSONデータ（message と timestamp）が表示された。
  • Q2: / はHTMLとして整形表示されたが、/api/hello はJSON形式のテキスト（波括弧で囲まれたデータ）がそのまま表示された。res.send() はHTMLを、res.json() はJSON形式のデータを返している。
  • Q3: 「Cannot GET /contact」というエラーメッセージが表示された。定義されていないURLにはExpressがデフォルトの404エラーを返す。
体験 コード体験 2 解答（記録例）
  • Q1: server.js のときは res.send() で返したHTML文字列が表示されたが、server2.js では public/index.html のスタイル付きHTMLページが表示された。express.static を使うと、HTMLファイルをそのまま配信できる。
  • Q2: {"server":"Express","staticDir":"public","message":"静的ファイルとAPIを同時に提供しています"} というJSONデータが表示された。
  • Q3: express.static('public') は、public フォルダ内のファイル（HTML, CSS, 画像など）をURLでアクセスできるように自動的に配信する仕組み。Live Server と同じように、ファイルをそのままブラウザに返す。

標準 標準課題 1 解答（サーバサイドの仕組み）

問1.

用語説明
Node.jsGoogle ChromeのV8エンジンを使い、ブラウザの外でJavaScriptを実行できるランタイム環境。サーバサイドプログラミングを可能にする。

問2. 解答例:

ブラウザが http://localhost:3000/about に対してGETリクエストを送る。Express がそのリクエストを受け取り、ルーティング機能で定義済みのルートを上から順に確認する。パスマッチングにより /about に一致するルートが見つかると、そのルートに紐づけられた処理関数が実行される。関数内で res.send() が呼ばれ、HTMLや文字列がレスポンスとしてブラウザに返される。

問3.

URL結果
http://localhost:3000/「Home」という文字列が表示される
http://localhost:3000/users[{"id":1,"name":"Sato"}] というJSON配列が表示される
http://localhost:3000/help「Help page」という段落がHTML形式で表示される
http://localhost:3000/login「Cannot GET /login」エラー（該当するルートが定義されていないため）

問4. 解答例:

res.send() は文字列やHTMLをそのまま返すメソッドで、Webページの表示に適している。res.json() はJavaScriptオブジェクトをJSON形式に変換して返すメソッドで、Content-Type ヘッダに application/json が自動で設定されるため、APIとしてデータを提供する場合に適している。
