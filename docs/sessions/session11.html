説明

第10回ではアバターの位置を同期しました。
今回はユーザの操作（クリック・インタラクション）を全ユーザ間で同期し、「誰かが箱をクリックしたら全員の画面で色が変わる」ような共有体験を実現します。

1.1 クリックイベントの同期 課題に出ます

教室のホワイトボードに誰かが文字を書くと、教室にいる全員がそれを見ることができます。
3D空間でも同じように、誰かがオブジェクトをクリックしたら、全員の画面でそのオブジェクトが変化する仕組みを作ります。

ユーザAがオブジェクトをクリックしたとき、以下の流れで全ユーザに反映されます。

A-Frame でクリックを検知するには、cursor コンポーネントを使います。

HTML
<!-- カーソル（視線 or マウス）を有効にする -->
<a-camera>
  <a-cursor color="white"></a-cursor>
</a-camera>

<!-- クリック対象のオブジェクト -->
<a-box id="target-box" class="clickable"
       position="0 1 -3" color="red"></a-box>

1.2 オブジェクトの状態管理 課題に出ます

Google ドキュメントでは、誰かが文章を編集すると全員の画面に即座に反映されます。さらに、後から開いた人も最新の状態のドキュメントを見ることができます。これはサーバが現在の状態を記憶しているからです。

3D空間のオブジェクトの状態（色、位置、回転など）をサーバ側で管理することで、以下の2つを実現します。

課題解決方法
リアルタイム反映クリック時にサーバで状態を更新し、全クライアントに配信する
途中参加新しいユーザが接続したとき、サーバが現在のオブジェクト状態を送信する
JavaScript（サーバ側の概念コード）
// サーバでオブジェクトの状態を管理
var objectStates = {
  'box-1': { color: 'red' },
  'box-2': { color: 'blue' },
  'box-3': { color: 'green' }
};

// 新規接続時: 現在の全状態を送信
socket.emit('initState', objectStates);

// クリック時: 状態を更新して全員に配信
socket.on('objectClick', function(data) {
  objectStates[data.id].color = data.newColor;
  io.emit('objectUpdated', data);
});

1.3 アニメーションの同期

A-Frame の animation コンポーネントを使い、全クライアントで同時にアニメーションを再生します。

サーバから「アニメーション開始」のイベントを受信したら、該当オブジェクトに setAttribute('animation', ...) でアニメーションを設定します。

JavaScript（クライアント側）
// クリック時: 回転アニメーションを設定
box.setAttribute('animation', {
  property: 'rotation',
  to: '0 360 0',
  dur: 1000,
  easing: 'easeInOutQuad'
});

1.4 共有オブジェクト vs 個人オブジェクト 課題に出ます

マルチユーザ3D空間では、オブジェクトを2種類に分けて考えます。

共有オブジェクト個人オブジェクト
定義全ユーザが同じ状態を見るオブジェクト各ユーザだけが見る・操作するオブジェクト
例部屋の家具、共有の掲示板、展示物自分のアバター、UIメニュー、カーソル
状態管理サーバで管理し、全員に同期する各クライアントのローカルで管理する
クリッククリックすると全員の画面で変化するクリックしても自分の画面だけ変わる

1.5 レイテンシへの対処の考え方

電話で会話すると、声がわずかに遅れて届くことがあります。ネットワーク通信でも同じ遅延（レイテンシ）が発生します。この遅延をゼロにはできませんが、ユーザが気にならないレベルに抑えることが重要です。
対策説明
楽観的更新サーバの応答を待たずに、クリックした本人の画面では即座に変化を反映する。サーバから確認が来なかった場合は元に戻す。
送信間隔の調整位置データの送信頻度を調整して、通信量と滑らかさのバランスを取る。
補間（Interpolation）受信した位置データ間を滑らかに補間して、カクカクした動きを軽減する（発展的な内容）。

例題

イベント同期のデータフロー

ユーザAが箱をクリックして色が変わるまでの流れを追います。

手順場所処理内容
1ユーザA（A-Frame）カーソルで箱をクリック → click イベントが発生
2ユーザA（JavaScript）クリックされた箱のIDと新しい色を socket.emit('objectClick', data) で送信
3サーバobjectStates の該当エントリを更新 → io.emit('objectUpdated', data) で全員に配信
4全クライアント受信したデータで該当オブジェクトの色を setAttribute('color', newColor) で変更

ポイント: 手順3でサーバが io.emit（全員）を使っています。クリックイベントではクリックした本人を含む全員に結果を送るため、broadcast.emit ではなく io.emit を使います。

コード体験 1

クリックで色が変わる共有オブジェクト

箱をクリックすると色が変わり、その変化が全ユーザの画面に反映される仕組みを作ります。

ファイル1: server.js

JavaScript ── server.js
var express = require('express');
var http = require('http');
var { Server } = require('socket.io');

var app = express();
var server = http.createServer(app);
var io = new Server(server);

app.use(express.static('public'));

// オブジェクトの状態を管理
var objectStates = {
  'box-1': { color: '#FF5252' },
  'box-2': { color: '#4FC3F7' },
  'box-3': { color: '#76B900' }
};

// クリック時に巡回する色リスト
var colorCycle = ['#FF5252', '#4FC3F7', '#76B900', '#FFB800', '#E040FB', '#FF6E40'];

io.on('connection', function(socket) {
  console.log('接続:', socket.id);

  // 新規接続時: 現在のオブジェクト状態を送信
  socket.emit('initState', objectStates);

  // オブジェクトクリック: 状態を更新して全員に配信
  socket.on('objectClick', function(data) {
    if (objectStates[data.id]) {
      // 次の色に巡回
      var currentIndex = colorCycle.indexOf(objectStates[data.id].color);
      var nextColor = colorCycle[(currentIndex + 1) % colorCycle.length];
      objectStates[data.id].color = nextColor;

      console.log('色変更:', data.id, '->', nextColor);
      io.emit('objectUpdated', { id: data.id, color: nextColor });
    }
  });

  socket.on('disconnect', function() {
    console.log('切断:', socket.id);
  });
});

server.listen(3000, function() {
  console.log('http://localhost:3000 でサーバ起動');
});

コード体験 2

クリックで回転アニメーションが同期される

箱をクリックすると回転アニメーションが再生され、その動きが全ユーザの画面で同時に実行されます。

ファイル1: server.js を以下で上書き

JavaScript ── server.js
var express = require('express');
var http = require('http');
var { Server } = require('socket.io');

var app = express();
var server = http.createServer(app);
var io = new Server(server);

app.use(express.static('public'));

// オブジェクトの状態を管理
var objectStates = {
  'obj-1': { color: '#FF5252', rotationY: 0, scale: 1 },
  'obj-2': { color: '#4FC3F7', rotationY: 0, scale: 1 },
  'obj-3': { color: '#76B900', rotationY: 0, scale: 1 }
};

var colorCycle = ['#FF5252', '#4FC3F7', '#76B900', '#FFB800', '#E040FB'];

io.on('connection', function(socket) {
  console.log('接続:', socket.id);

  // 新規接続時: 現在の状態を送信
  socket.emit('initState', objectStates);

  // オブジェクトクリック: 色変更 + 回転 + スケール変更
  socket.on('objectClick', function(data) {
    var state = objectStates[data.id];
    if (!state) return;

    // 色を次に巡回
    var idx = colorCycle.indexOf(state.color);
    state.color = colorCycle[(idx + 1) % colorCycle.length];

    // 回転を 90度ずつ加算
    state.rotationY = state.rotationY + 90;

    // スケールをトグル (1 → 1.5 → 1)
    state.scale = state.scale === 1 ? 1.5 : 1;

    console.log('更新:', data.id, state);
    io.emit('objectUpdated', {
      id: data.id,
      color: state.color,
      rotationY: state.rotationY,
      scale: state.scale
    });
  });

  socket.on('disconnect', function() {
    console.log('切断:', socket.id);
  });
});

server.listen(3000, function() {
  console.log('http://localhost:3000 でサーバ起動');
});

    <a-plane position="0 0 0" rotation="-90 0 0"

    var socket = io();

    // 初期状態の受信
    socket.on('initState', function(states) {
      for (var id in states) {
        var el = document.getElementById(id);
        if (el) {
          el.setAttribute('color', states[id].color);
          el.setAttribute('rotation', '0 ' + states[id].rotationY + ' 0');
          var s = states[id].scale;
          el.setAttribute('scale', s + ' ' + s + ' ' + s);
        }
      }

    // クリック対象にイベントリスナを設定

        easing: 'easeOutElastic'
      });
    });

  • Q1: 赤い箱をクリックすると水色に変わった。もう一度クリックすると緑に変わった。色がリスト順に巡回していた（赤 → 水色 → 緑 → 黄 → 紫 → オレンジ → 赤...）。

体験 コード体験 2 解答（記録例）

標準課題の解答

標準 標準課題 1 解答（イベント同期の仕組み）

問1. 解答例:

ユーザAがA-Frame空間内で箱をクリックすると、ブラウザで click イベントが発生する。そのイベントハンドラ内で socket.emit('objectClick', { id: ... }) を呼び出し、クリックされた箱のIDをサーバに送信する。サーバは受信したIDをもとに objectStates の該当エントリの色を次の色に更新する。その後、サーバは io.emit('objectUpdated', data) で更新後の色を全クライアントに配信する。各クライアントは受信したデータで setAttribute('color', newColor) を呼び出し、箱の色を変更する。

問2.

共有オブジェクト個人オブジェクト
定義全ユーザが同じ状態を見るオブジェクト各ユーザだけが見る・操作するオブジェクト
具体例部屋の家具、展示物、掲示板自分のアバター、UIメニュー、カーソル
状態管理の場所サーバで管理し、全員に同期する各クライアントのローカルで管理する

問3. 解答例:

サーバが objectStates でオブジェクトの状態を管理する理由は、途中参加のユーザに最新の状態を送るためである。もし管理しなかった場合、途中から接続したユーザの画面では全ての箱が初期色のままとなり、既に接続していたユーザとは異なる色の箱を見ることになる。つまり、ユーザ間で3D空間の見え方に矛盾が生じてしまう。

問4. 解答例:

クリック処理（io.emit）: クリックの結果（色変更）はクリックした本人にも反映する必要がある。サーバが色の巡回を決定するため、本人もサーバからの結果を受け取って反映する必要がある。
アバター移動（broadcast.emit）: 自分のアバターを自分の画面に表示する必要はない（自分のカメラ視点で見ているため）。自分に送ってしまうと、自分のカメラ位置にアバターが表示されてしまい、邪魔になる。

発展 発展課題 解答例（マルチユーザ・ギャラリー）

テーマ: 「カラフル・ジオメトリ・ギャラリー」

server.js:

var express = require('express');
var http = require('http');
var { Server } = require('socket.io');

var app = express();
var server = http.createServer(app);
var io = new Server(server);

app.use(express.static('public'));

var objectStates = {
  'art-1': { color: '#FF5252', rotationY: 0, scale: 1 },
  'art-2': { color: '#4FC3F7', rotationY: 0, scale: 1 },
  'art-3': { color: '#76B900', rotationY: 0, scale: 1 },
  'art-4': { color: '#E040FB', rotationY: 0, scale: 1 },
  'art-5': { color: '#FFB800', rotationY: 0, scale: 1 }
};

var colorCycle = ['#FF5252', '#4FC3F7', '#76B900', '#FFB800', '#E040FB', '#FF6E40'];

io.on('connection', function(socket) {
  socket.emit('initState', objectStates);

  socket.on('objectClick', function(data) {
    var state = objectStates[data.id];
    if (!state) return;
    var idx = colorCycle.indexOf(state.color);
    state.color = colorCycle[(idx + 1) % colorCycle.length];
    state.rotationY = state.rotationY + 90;
    state.scale = state.scale === 1 ? 1.3 : 1;

    io.emit('objectUpdated', {
      id: data.id,
      color: state.color,
      rotationY: state.rotationY,
      scale: state.scale
    });
  });

  socket.on('disconnect', function() {});
});

server.listen(3000, function() {
  console.log('http://localhost:3000');
});
