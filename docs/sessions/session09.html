1.1 HTTPの限界 課題に出ます

HTTPは「手紙のやりとり」です。クライアントが手紙（リクエスト）を送らない限り、サーバから連絡することはできません。チャットのように「相手が発言したら即座に届く」仕組みには向いていません。

第1回で学んだ HTTP通信 はリクエスト - レスポンス型です。

  • クライアントがリクエストを送る → サーバがレスポンスを返す → 通信終了
  • サーバ側から自発的にクライアントへデータを送ることはできない
  • 新しい情報を得るには、クライアントが繰り返しリクエストするしかない（ポーリング）

チャット、オンラインゲーム、共同編集などリアルタイム性が求められるアプリには、HTTPだけでは不十分です。

1.2 WebSocket とは 課題に出ます

HTTPが「手紙」なら、WebSocket は「電話」です。一度つながれば、どちらからでもいつでも話しかけられる常時接続の通信路です。

WebSocket は、クライアントとサーバの間に永続的な双方向通信路を開くプロトコルです。

HTTPWebSocket
通信方向クライアント → サーバのみ（一方向）双方向（どちらからでも送信可能）
接続毎回接続→切断の繰り返し一度接続したら維持し続ける
リアルタイム性低い（ポーリングが必要）高い（即座にデータが届く）
用途Webページの表示、API通信チャット、ゲーム、共同編集

1.3 Socket.IO とは 課題に出ます

Socket.IO は、WebSocket を簡単に使えるようにしたライブラリです。

機能説明
自動フォールバックWebSocket が使えない環境では、HTTP ロングポーリングに自動で切り替える
自動再接続接続が切れても自動で再接続を試みる
イベントベースemit（送信）と on（受信）の2つのメソッドだけで通信できる
ルーム特定のグループにだけメッセージを送れる（第10回以降で使用）

Socket.IO はサーバ側（Node.js）とクライアント側（ブラウザ）の両方にライブラリがあり、セットで使います。

1.4 emit / on パターン コード体験に直結

emit は「叫ぶ」、on は「耳を傾ける」です。Aさんが emit('hello', 'こんにちは') で叫ぶと、on('hello', ...) で耳を傾けている全員に届きます。

Socket.IO の通信はイベント名とデータのペアで行います。

JavaScript ── 基本パターン
// 送信側（どちら側からでも使える）
socket.emit('イベント名', データ);

// 受信側（どちら側からでも使える）
socket.on('イベント名', function(データ) {
  // データを受け取ったときの処理
});

1.5 broadcast（ブロードキャスト）課題に出ます

サーバからメッセージを送る方法は3種類あります。

メソッド送信先使い場面
io.emit('event', data)全員（送信者を含む全クライアント）全体チャットメッセージ
socket.broadcast.emit('event', data)送信者以外の全クライアント「○○さんが入室しました」通知
socket.emit('event', data)送信者のみエラーメッセージ、個別通知

1.6 接続/切断イベント コード体験に直結

Socket.IO には組み込みのイベントがあります。

JavaScript ── サーバ側
io.on('connection', function(socket) {
  console.log('ユーザが接続しました', socket.id);

  socket.on('disconnect', function() {
    console.log('ユーザが切断しました', socket.id);
  });
});
イベント発生タイミング引数
connectionクライアントがサーバに接続したときsocket（その接続を表すオブジェクト）
disconnectクライアントが切断したとき（タブを閉じるなど）切断理由の文字列

socket.id は各接続に自動で割り当てられる一意のIDです。ユーザの識別に使えます。

Socket.IO の接続確認 ── 最小構成

まず、サーバとクライアントが接続できることを確認します。2つのファイルを作成してください。

ファイル1: server.js（Web09 フォルダ直下に作成）

JavaScript ── server.js
const express = require('express');
const http = require('http');
const { Server } = require('socket.io');

const app = express();
const server = http.createServer(app);
const io = new Server(server);

// public フォルダの中身を静的ファイルとして配信
app.use(express.static('public'));

// クライアントが接続したとき
io.on('connection', function(socket) {
  console.log('接続:', socket.id);

  // クライアントが切断したとき
  socket.on('disconnect', function() {
    console.log('切断:', socket.id);
  });
});

// サーバをポート3000で起動
server.listen(3000, function() {
  console.log('サーバ起動: http://localhost:3000');
});

テキストチャットアプリを作る

接続確認ができたら、チャット機能を作ります。server.js と public/index.html の両方を書き換えてください。

重要: まずターミナルで Ctrl+C を押してサーバを停止してから、コードを書き換えてください。

ファイル1: server.js（上書き）

JavaScript ── server.js
const express = require('express');
const http = require('http');
const { Server } = require('socket.io');

const app = express();
const server = http.createServer(app);
const io = new Server(server);

app.use(express.static('public'));

io.on('connection', function(socket) {
  console.log('接続:', socket.id);

  // クライアントから 'chat' イベントを受け取ったとき
  socket.on('chat', function(msg) {
    console.log('メッセージ:', msg);
    // 全クライアントに転送する
    io.emit('chat', msg);
  });

  socket.on('disconnect', function() {
    console.log('切断:', socket.id);
  });
});

server.listen(3000, function() {
  console.log('サーバ起動: http://localhost:3000');
});

      var msg = input.value;
      if (msg) {
        socket.emit('chat', msg);
        input.value = '';
      }
    });

    // Enterキーでも送信
    input.addEventListener('keydown', function(e) {
      if (e.key === 'Enter') {
        sendBtn.click();
      }
    });

    // サーバからメッセージを受け取ったとき
    socket.on('chat', function(msg) {
      var li = document.createElement('li');
      li.textContent = msg;
      messages.appendChild(li);
      // 自動スクロール
      messages.scrollTop = messages.scrollHeight;
    });

トレース トレース演習 1 解答
順番何が起きるか誰が受け取るか
①ユーザAが emit('chat', 'hello') を実行サーバ
②サーバの on('chat', ...) が発火し、msg に 'hello' が入るサーバ
③サーバが io.emit('chat', msg) を実行ユーザA + ユーザB（全員）
④ユーザAとユーザBの on('chat', ...) が発火ユーザA + ユーザB

追加問題の解答: socket.broadcast.emit を使った場合、③の送信先はユーザBのみ（送信者Aを除いた全員）になる。ユーザAの画面にはメッセージが表示されない。

トレース トレース演習 2 解答
コード何をしているか
require('express')Express フレームワークを読み込む。Webサーバの機能（静的ファイル配信、ルーティング等）を提供する。
http.createServer(app)Express アプリを使ってHTTPサーバを作成する。Socket.IO を同じサーバ上で動かすために必要。
new Server(server)HTTPサーバにSocket.IO のリアルタイム通信機能を追加する。
express.static('public')public フォルダの中身（HTML, CSS, JS, 画像など）を静的ファイルとしてブラウザに配信する設定。
server.listen(3000)サーバをポート3000で起動し、クライアントからの接続を待ち受ける。
体験 コード体験 1 解答（記録例）
  • Q1: 黒い背景に「Socket.IO 接続テスト」という見出しと、「接続成功! ID: _abc123def」のような緑色のテキストが表示された。
  • Q2: ブラウザのタブを閉じると、ターミナルに「切断: _abc123def」のようなログが出た。
  • Q3: タブを2つ開くと、ターミナルに「接続: xxx」のログが2行出た。それぞれのIDは異なっていた（各接続に固有のIDが割り当てられる）。
体験 コード体験 2 解答（記録例）
  • Q1: タブ1で送ったメッセージが、タブ2にもリアルタイムで表示された。
  • Q2: タブ2で送ったメッセージも、タブ1にリアルタイムで表示された。双方向通信ができている。
  • Q3: ターミナルに「メッセージ: こんにちは」のように、送信されたメッセージの内容がログとして出力された。
  • Q4: 「切断: xxxxx」のようなログが出て、そのユーザのIDが表示された。
