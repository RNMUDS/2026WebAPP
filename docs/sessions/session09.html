1.1 HTTP と WebSocket の違い

HTTP は「手紙」、WebSocket は「電話」です。手紙はこちらが送らないと返事が来ませんが、電話は一度つながればどちらからでもいつでも話しかけられます。チャットには電話＝WebSocket が向いています。

第1回で学んだ HTTP はリクエスト→レスポンス型で、クライアントが聞きに行かないとサーバからデータは来ません。新着を知るには何度も聞き直す必要があり、チャットには不向きです。

WebSocket は、クライアントとサーバの間につなぎっぱなしの通信路を開き、どちらからでもデータを送れるようにする仕組みです。

HTTPWebSocket
通信方向クライアント → サーバのみ双方向（どちらからでも送れる）
接続毎回 接続→切断 の繰り返し一度つないだら維持し続ける
リアルタイム性低い高い（すぐ届く）
用途Webページ表示、API通信チャット、ゲーム、共同編集

1.2 Socket.IO とは

Socket.IO（ソケット・アイオー）は、WebSocket を簡単に使えるようにしたライブラリです。サーバ側（Node.js）とクライアント側（ブラウザ）の両方にライブラリがあり、セットで動かします。

通信は emit（送信）と on（受信）のたった2つで行います。これが今回いちばん大事な考え方です。👇

1.3 emit / on パターン この回のハイライト

emit は「叫ぶ」、on は「耳を傾ける」です。Aさんが emit('chat', 'こんにちは') とイベント名つきで叫ぶと、同じ 'chat' という名前で on('chat', ...) と耳を傾けている人にだけ届きます。

Socket.IO の通信は、すべて「イベント名」と「データ」の2点セットで行われます。まずは emit と on の形（部品）を分解して見てみましょう。

ポイント： emit と on はイベント名（上の例なら 'chat'）が一致したペアだけがつながります。名前を変えれば、用途ごとに通信を分けられます（例：'chat' はメッセージ、'join' は入室）。

▶ メッセージが全員に届くまで（アニメーション）

「こんにちは」が Aの画面 → サーバ → 全員の画面 へ流れる様子を、1ステップずつ追いかけます。

※ 送信者A自身にも表示されるのは、サーバが io.emit（＝全員）で配信しているためです。

上のアニメーションをコードにすると：

JavaScript ── クライアント（送信側）
// ② 送信ボタンで emit する
socket.emit('chat', 'こんにちは');

// ⑤ 'chat' が届いたら画面に表示する
socket.on('chat', function(msg) {
  console.log('受信:', msg);  // msg = 'こんにちは'
});
JavaScript ── サーバ（中継役）
io.on('connection', function(socket) {
  // ③ クライアントから 'chat' を受け取ったら…
  socket.on('chat', function(msg) {
    // ④ 全員へ転送する
    io.emit('chat', msg);
  });
});
JavaScript ── 基本パターン（まとめ）
// 送信側（サーバ・クライアントどちらでも使える）
socket.emit('イベント名', データ);

// 受信側（サーバ・クライアントどちらでも使える）
socket.on('イベント名', function(データ) {
  // データを受け取ったときの処理
});

✅ ここだけ覚えれば OK

  • emit = 送る、on = 受け取る
  • 必ずイベント名（例 'chat'）とデータの2点セット
  • emit と on は同じイベント名どうしがつながる
  • サーバの io.emit は全員へ配信（自分にも届く）

1.4 メッセージの送り先（broadcast）

サーバからメッセージを送る方法は主に3種類あります。送り先を切り替えられるのがポイントです。

メソッド送信先使い場面
io.emit('event', data)全員（送信者を含む）通常のチャットメッセージ
socket.broadcast.emit('event', data)送信者以外の全員「○○さんが入室しました」通知
socket.emit('event', data)送信者のみ本人だけへの個別通知

io は全接続の管理者、socket は1つの接続を表します。この違いで送り先が変わると覚えておきましょう。

1.5 接続 / 切断イベント

Socket.IO には組み込みのイベントがあります。

JavaScript ── サーバ側
io.on('connection', function(socket) {
  console.log('ユーザが接続しました', socket.id);

  socket.on('disconnect', function() {
    console.log('ユーザが切断しました', socket.id);
  });
});
イベント発生タイミング
connectionクライアントが接続したとき（socket を受け取る）
disconnectクライアントが切断したとき（タブを閉じるなど）

socket.id は各接続に自動で割り当てられる一意のIDです（タブごとに別のID、リロードでも変わります）。

📖 キーワードまとめ

用語意味
WebSocketクライアントとサーバ間につなぎっぱなしの双方向通信路を開く仕組み。
Socket.IOWebSocket を簡単に使えるライブラリ。emit/on で通信する。
emit / onemit はデータ送信、on は受信の待ち受け。
broadcast送信者以外の全員に送る方法。入退室の通知などに使う。
socket.id各接続に自動で割り当てられる一意のID。

Socket.IO の通信フローを読む

以下のサーバコードとクライアントコードを見て、通信の流れを確認しましょう。

JavaScript ── サーバ（server.js）
io.on('connection', function(socket) {
  // 'chat' が届いたら、全員へ転送する

    io.emit('chat', msg);
  });

JavaScript ── クライアント（index.html内）
// サーバへ送信
socket.emit('chat', 'こんにちは');

// サーバから届いたら表示

  console.log('受信:', msg);
});
順序何が起きるか
1クライアントが emit('chat', 'こんにちは') でサーバに送信
2サーバの on('chat', ...) が発火し msg = 'こんにちは' を受け取る
3サーバが io.emit('chat', msg) で全クライアントへ送信
4各クライアントの on('chat', ...) が発火して表示

まずは接続できるか確認する

JavaScript ── server.js
const express = require('express');
const http = require('http');
const { Server } = require('socket.io');

const app = express();
const server = http.createServer(app);
const io = new Server(server);

// public フォルダの中身を配信
app.use(express.static('public'));

io.on('connection', function(socket) {
  console.log('接続:', socket.id);
  socket.on('disconnect', function() {
    console.log('切断:', socket.id);
  });
});

server.listen(3000, function() {
  console.log('サーバ起動: http://localhost:3000');
});

コード体験 2

チャットアプリを完成させる 🎯

接続が確認できたら、いよいよチャットにします。さきほどの2ファイルを、次の内容に書き換えてください。

① server.js ── chat を受け取ったら全員へ転送する

JavaScript ── server.js
const express = require('express');
const http = require('http');
const { Server } = require('socket.io');

const app = express();
const server = http.createServer(app);
const io = new Server(server);

app.use(express.static('public'));

io.on('connection', function(socket) {
  // クライアントから 'chat' が届いたら…
  socket.on('chat', function(msg) {
    // 全員（自分を含む）へ転送する
    io.emit('chat', msg);
  });
});

server.listen(3000, function() {
  console.log('サーバ起動: http://localhost:3000');
});

② public/index.html ── 入力欄・送信ボタン・メッセージ表示

HTML ── public/index.html
<!DOCTYPE html>
<html lang="ja">
<head>
  <meta charset="UTF-8">
  <title>チャット</title>
  <style>
    body { font-family: sans-serif; background: #111; color: #eee; padding: 1.5rem; }
    #messages { list-style: none; padding: 0; }
    #messages li { background: #1e1e1e; padding: 0.5rem 0.8rem; border-radius: 8px; margin-bottom: 0.4rem; }
    input { padding: 0.5rem; width: 60%; }
    button { padding: 0.5rem 1rem; }
  </style>
</head>
<body>
  <h1>チャット</h1>
  <ul id="messages"></ul>
  <input id="input" placeholder="メッセージを入力">
  <button id="send">送信</button>

  <script src="/socket.io/socket.io.js"></script>
  <script>
    var socket = io();
    var input = document.getElementById('input');
    var messages = document.getElementById('messages');

    // 送信ボタンで 'chat' を送る
    document.getElementById('send').addEventListener('click', function() {
      socket.emit('chat', input.value);
      input.value = '';
    });

    // 'chat' を受け取ったら画面に追加する
    socket.on('chat', function(msg) {
      var li = document.createElement('li');
      li.textContent = msg;
      messages.appendChild(li);
    });
  </script>
</body>
</html>

例題 例題 Socket.IO の通信フローを読む 解答

サーバ側の io.on('connection', ...) はクライアントが接続するたびに呼ばれ、その接続に対応する socket オブジェクトを受け取ります。クライアントが emit('chat', 'こんにちは') を実行すると、サーバの socket.on('chat', ...) が発火し、その中で io.emit('chat', msg) を呼ぶことで全クライアントに同じメッセージが転送されます。

順序場所処理内容
1クライアントsocket.emit('chat', 'こんにちは') で送信
2サーバsocket.on('chat', ...) が発火、msg = 'こんにちは'
3サーバio.emit('chat', msg) で全クライアントへブロードキャスト
4全クライアント各 socket.on('chat', ...) が発火し画面に表示

ポイント: socket（小文字）は個別の接続、io は全接続の管理者。送信先を変えたいときはこの2つを使い分ける。

例題 1.4 やってみよう（io.emit と socket.broadcast.emit の違い）解答

io.emit('chat', msg) は送信者を含む全クライアントに届くのに対し、socket.broadcast.emit('chat', msg) は送信者以外の全クライアントにだけ届く。書き換えた状態でメッセージを送ると、自分の画面にはメッセージが表示されず、他のタブにだけ表示される。

// 全員に届く（自分にも届く）
io.emit('chat', msg);

// 自分以外の全員に届く（自分には届かない）
socket.broadcast.emit('chat', msg);

「○○さんが入室しました」のような本人には不要な通知には broadcast が便利。一方、通常のチャットでは自分の発言も自分の画面に出したいので io.emit を使う。

トレース トレース演習 1 解答
順番何が起きるか誰が受け取るか
①ユーザAが emit('chat', 'hello') を実行サーバ
②サーバの on('chat', ...) が発火し、msg に 'hello' が入るサーバ
③サーバが io.emit('chat', msg) を実行ユーザA + ユーザB（全員）
④ユーザAとユーザBの on('chat', ...) が発火ユーザA + ユーザB

追加問題の解答: socket.broadcast.emit を使った場合、③の送信先はユーザBのみ（送信者Aを除いた全員）になる。ユーザAの画面にはメッセージが表示されない。
