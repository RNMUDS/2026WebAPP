授業概要

本授業では、Webアプリケーションを構成する技術（サーバ・3D表現・リアルタイム通信・データベース）を段階的に学習し、実装します。

Webの仕組みは、ショッピングサイト・SNS・オンラインゲーム・地図アプリなど、日常で使うほぼすべてのサービスの土台です。この授業では、自分でWebサーバを立て、ブラウザで動くアプリケーションを作れるようになることを目指します。

後半では A-Frame による3D表現や Socket.IO によるリアルタイム通信も扱い、3Dマルチユーザ空間の構築にも挑戦します。

到達目標

  • HTTP通信の仕組みとクライアント・サーバモデルを説明できる
  • Node.js + Express.js でWebサーバを構築し、ルーティング・API設計ができる
  • A-Frame を使って3D空間をブラウザ上に構築できる
  • Socket.IO でリアルタイム通信を実装し、マルチユーザ体験を作れる
  • 企画・設計・実装・発表の一連の開発プロセスを経験する

説明

普段使っているWebサイトの裏側で何が起きているかを理解します。
コードを書くことが目的ではなく、仕組みを言葉と図で説明できるようになることがゴールです。

1.1 HTTP通信 ── 手紙のやりとりのイメージ

あなた（ブラウザ）が郵便局（インターネット）を通じて、お店（サーバ）に注文書（リクエスト）を送る。
お店は注文を受け取り、商品（レスポンス）を送り返す。この「注文と配達」の手順がHTTP通信です。

ブラウザでURLを入力して Enter を押すたびに、この「手紙のやりとり」が起きています。

1.2 クライアントとサーバ ── レストランのイメージ

レストランでは、お客さん（クライアント）がメニューを見て注文し、厨房（サーバ）が料理を作って届けます。
Webでは、ブラウザがお客さん、Node.js が厨房の役割です。

Node.js は、JavaScript をブラウザの外（サーバ側）で動かすためのランタイムです。Google Chrome の V8 エンジンがベースで、npm というパッケージマネージャで機能を拡張できます。

1.3 Express.js ── レストランのウェイター

レストランにウェイターがいると、注文の受付・料理の配膳がスムーズになります。
Express.js は Node.js の「ウェイター」です。URLに応じた処理の振り分け（ルーティング）を簡単に書けます。

Node.js だけでもサーバは作れますが、Express.js を使うと:

  • ルーティング（URLに応じて処理を振り分け）が簡単に書ける
  • 静的ファイル配信（HTML・CSS・画像など）が1行でできる
  • ミドルウェアで機能を柔軟に追加できる

1.4 ルーティングの全体像

Express がリクエストを受け取ってからレスポンスを返すまでの流れです。

用語意味コード例
ルーティングURLに応じて処理を振り分けることapp.get('/about', ...)
リクエスト (req)ブラウザからの要求情報req.query, req.params
レスポンス (res)サーバからの返答res.send(), res.json()
静的ファイル配信HTML/CSS/画像をそのまま返すapp.use(express.static('public'))

ルーティングの動作を追う

以下の Express サーバが動いているとき、ブラウザで各URLにアクセスした結果を追ってみましょう。

JavaScript
app.get('/', (req, res) => {

app.get('/about', (req, res) => {

const app = express();
const PORT = 3000;

// トップページ

app.get('/about', (req, res) => {
  res.send('<h1>このサイトについて</h1><p>Express.jsで作った最初のサーバです。</p><a href="/">トップに戻る</a>');

// JSON API
app.get('/api/info', (req, res) => {

const express = require('express');
const app = express();
const PORT = 3000;

// トップページ
app.get('/', (req, res) => {
  res.send(`
    <!DOCTYPE html>
    <html lang="ja">
    <head><meta charset="UTF-8"><title>My Profile</title></head>
    <body>
      <h1>山田太郎のプロフィール</h1>
      <p>Webプログラミングの授業ページです。</p>
      <a href="/about">自己紹介ページ</a><br>
      <a href="/api/profile">プロフィール API (JSON)</a>
    </body>
    </html>
  `);
});

// 自己紹介ページ
app.get('/about', (req, res) => {
  res.send(`
    <h1>自己紹介</h1>
    <p>名前: 山田太郎</p>
    <p>学年: 3年</p>
    <a href="/">トップに戻る</a>
  `);
});

// プロフィール API
app.get('/api/profile', (req, res) => {
  res.json({
    name: '山田太郎',
    grade: 3,
    interests: ['Web', '3D', 'ゲーム']
  });
});

app.listen(PORT, () => {
  console.log(`サーバ起動: http://localhost:${PORT}`);
});

  • /about のルートで自分の名前・学年がHTMLで返される
