ガイダンス

授業概要

本授業では、Webプログラミングの基礎から応用までを体系的に学び、最終的には複数の利用者が同時に参加できるインタラクティブなWebアプリケーションを制作します。

サーバサイドプログラミング・クライアントとサーバ間の通信・3D表現・リアルタイム通信を段階的に学びます。最終的に A-Frame + Node.js + Socket.IO を組み合わせ、マルチプレイ・チャット機能を備えたWebXRアプリを制作・発表します。

この授業のゴール

15回の授業を通じて、以下のようなマルチユーザ3D Webアプリケーションを作れるようになります。

  • A-Frame でブラウザ上に3D空間を構築
  • Node.js + Express でサーバを構築
  • Socket.IO で複数ユーザがリアルタイムに接続
  • 3D空間内でアバターが動き、吹き出しチャットで会話できる
  • スマートフォンやVRヘッドセットでも体験可能（WebXR）

毎回の演習で学ぶ技術が、最終制作のパーツになります。

到達目標

  • Webアプリケーションの基本構造（クライアント・サーバ）を説明できる
  • サーバサイドプログラムを用いてWebアプリケーションを構築できる
  • クライアントとサーバ間でデータ通信を行うことができる
  • データベースと連携したWebアプリケーションを構築できる
  • リアルタイム性を持つWebアプリケーションの基礎を理解する
  • 複数人が利用するインタラクティブなWebアプリケーションを設計・実装できる

Visual Studio Code（VS Code）とは

Visual Studio Code（以下 VS Code）は、Microsoft が無料で提供しているコードエディタです。HTMLやJavaScriptなどのコードを書くための専用アプリで、この授業ではすべてのコード作成・実行を VS Code 上で行います。

VS Code には拡張機能と呼ばれるプラグインを追加でき、機能を自由に拡張できます。今日はLive Server という拡張機能を使い、書いたHTMLをすぐにブラウザで確認できる環境を整えます。

教室PCの場合: VS Code はすでにインストールされています。デスクトップまたはスタートメニューから起動してください。
Step 1

Live Server 拡張機能をインストールする

  1. VS Code を起動する
  2. 左サイドバーの拡張機能アイコン（四角が4つ並んだマーク）をクリック
  3. 検索欄に Live Server と入力
  4. 「Live Server」（作者: Ritwick Dey）の「インストール」ボタンをクリック
Live Server とは: HTMLファイルをローカルWebサーバで配信し、ブラウザで即座にプレビューできる拡張機能です。ファイルを保存するたびに自動でブラウザが更新されます。
Step 2

作業フォルダを作成して開く

  1. Sドライブに Web_Prog という名前のフォルダを作成する（授業全体の作業フォルダ）
  2. その中に Web01 というフォルダを作成する（今回の作業フォルダ）
  3. VS Code で「ファイル」→「フォルダーを開く」から S:\Web_Prog\Web01 を開く
フォルダ構成: 毎回 Web_Prog の中に Web01, Web02, ... とフォルダを分けて管理します。フォルダ名は半角英数字で、日本語やスペースは使わないでください。
Step 3

HTMLファイルを作成してプレビューする

  1. VS Code のエクスプローラ（左パネル）で新しいファイルを作成し、ファイル名を index.html にする
  2. このページのコードブロック右上にある「コピー」ボタンでコードをコピーし、貼り付ける
  3. ファイルを保存する（Ctrl+S / Cmd+S）
  4. エディタ上で右クリック → 「Open with Live Server」を選択
  5. ブラウザが自動で開き、3D空間が表示される
ポイント: ファイルを保存するだけでブラウザが自動更新されます。コードを変えて保存 → すぐに結果を確認できます。

1.1 クライアントとサーバ

レストランでは、お客さんがメニューを見て注文し、厨房が料理を作って届けます。
Webでも同じように、「要求する側」と「応答する側」に分かれています。
用語役割具体例
クライアントサービスを要求する側。ユーザが直接操作する。Webブラウザ（Chrome, Safari, Edge など）、スマートフォンアプリ
サーバサービスを提供する側。クライアントからの要求を受け取り、処理して返答する。24時間稼働し、複数のクライアントに同時に対応する。Webサーバ（Apache, Nginx）、アプリケーションサーバ（Node.js, Python など）

普段Webサイトを見るとき、あなたのブラウザ（クライアント）は世界のどこかにあるサーバにアクセスしてページのデータを受け取っています。1台のサーバに、何千・何万ものクライアントが同時にアクセスすることもあります。

1.2 HTTP / HTTPS通信 ── クライアントとサーバの会話のルール

日本語で会話するには日本語の文法が必要なように、クライアントとサーバが会話するにもルール（プロトコル）が必要です。そのルールが HTTP です。

HTTP（HyperText Transfer Protocol）は、クライアントとサーバの通信手順を定めたプロトコル（約束事）です。すべてのWeb通信はこのルールに従います。

用語説明
リクエストクライアントからサーバへの要求。「どのページが欲しいか（URL）」と「何をしたいか（メソッド）」を伝える。
レスポンスサーバからクライアントへの返答。「処理結果（ステータスコード）」と「データ本体（HTML, 画像, JSON など）」を返す。
ステータスコード200=成功、404=ページが見つからない、500=サーバ内部エラー

HTTP と HTTPS の違い

HTTPS は HTTP に暗号化（TLS）を加えたものです。通信内容が暗号化されるため、パスワードやクレジットカード情報を安全に送れます。現在のWebサイトはほぼすべて HTTPS を使用しており、ブラウザのURLバーに鍵マークが表示されます。

今日の Live Server も同じ仕組みです。VS Code が http://127.0.0.1:5500 でWebサーバを起動し、ブラウザがHTTPリクエストを送ってHTMLを受け取っています。

1.3 Webアプリケーションとは

Webサイトが「掲示板に貼られたポスター」なら、Webアプリケーションは「窓口で対話しながら手続きできる銀行ATM」です。見るだけでなく、操作して反応が返ってきます。
静的WebサイトWebアプリケーション
動作あらかじめ用意されたHTML/CSS/画像をそのまま返すサーバ側でプログラムが動き、リクエストに応じて異なるレスポンスを生成する
例企業紹介ページ、ブログ、ポートフォリオSNS、ショッピングサイト、Google Maps、チャットアプリ
サーバの役割ファイルをそのまま配信するだけユーザの入力を処理し、データベースの読み書き、リアルタイム通信などを行う

この授業で作るのはWebアプリケーションです。サーバ側でプログラムを書き、ユーザの操作に応じてリアルタイムに反応するシステムを構築します。

1.4 Webアプリケーションの実装方法

Webアプリケーションのサーバ側は、さまざまなプログラミング言語・フレームワークで構築できます。

言語フレームワーク特徴
JavaScriptNode.js + Expressブラウザ側と同じ言語でサーバも書ける。リアルタイム通信に強い。← この授業で使用
PythonFastAPI / Django / Flask読みやすい文法。AI・データ分析との連携に強い。
RubyRuby on Rails高速な開発。スタートアップで人気。
PHPLaravelWebに特化。WordPress の基盤。
Go / JavaGin / Spring Boot大規模サービスに適した高性能言語。

なぜ Node.js を使うのか？

ブラウザ側（クライアント）の言語は JavaScript です。Node.js を使えば、サーバ側も同じ JavaScript で書けるため、1つの言語だけでWebアプリ全体を構築できます。また、Socket.IO によるリアルタイム通信との相性が良く、マルチプレイ機能の実装に適しています。

1.5 WebXR ── ブラウザで3D・VR・AR体験を届ける技術

VRゲームを遊ぶには専用アプリのダウンロードが必要です。でもWebXRなら、URLにアクセスするだけで3D空間やVR体験をブラウザ上で楽しめます。

XR は VR（仮想現実）、AR（拡張現実）、MR（複合現実）をまとめた総称です。WebXR はこれらの体験をWebブラウザ上で実現するための技術仕様で、以下の特徴があります。

  • インストール不要 ── URLを共有するだけで誰でも体験できる
  • マルチデバイス対応 ── PC、スマートフォン、VRヘッドセット（Meta Quest 等）のどれでも動く
  • Web技術で開発 ── HTML + JavaScript で作れる。UnityやUnreal Engineは不要

この授業のゴールは、WebXR技術を使って3Dマルチユーザ空間を構築することです。

1.6 A-Frame ── HTMLタグで3D空間を作るフレームワーク

普通のWebページは <h1> や <p> でテキストを表示します。A-Frame を使うと <a-box> や <a-sphere> といった独自タグで3Dオブジェクトをブラウザ上に表示できます。

A-Frame は Mozilla が開発した WebXR フレームワークです。内部では Three.js（JavaScript の 3D描画ライブラリ）が動いていますが、開発者はHTMLタグを書くだけで3Dシーンを構築できます。公式ドキュメント: aframe.io/docs

普通のHTMLA-Frame（3D用HTML）
<h1> → 見出しテキスト<a-box> → 3Dの箱
<img> → 画像<a-sphere> → 3Dの球
<div> → 四角い領域<a-scene> → 3D空間全体

A-Frameの基本ルール:

  • すべての3Dオブジェクトは <a-scene> の中に書く
  • position="x y z" で位置を指定する（x: 左右、y: 上下、z: 手前奥）
  • color で色、radius で半径など、属性で見た目を制御する
  • CDN から1行で読み込むだけで使える（インストール不要）

順番何が起きるか
①VS Code が http://127.0.0.1:5500 で小さな Webサーバ を起動する
②ブラウザが自動で開き、サーバに HTTPリクエスト を送る
③サーバが index.html ファイルを読み、レスポンス としてブラウザに返す
④ブラウザがHTMLを解析し、<script> を見つけて A-Frame ライブラリを ダウンロード（読み込み） する
⑤A-Frame が <a-scene> 内のタグを読み、3D空間 を描画する
体験 コード体験 1 解答（記録例）
  • Q1: 赤い箱、緑色の床、水色の空が表示された
  • Q2: マウスでドラッグすると視点が回転し、箱を別の角度から見ることができた。WASDキーでカメラの位置を移動できた。
体験 コード体験 2 解答（記録例）
  • Q1: トレース演習1の予測とほぼ合っていた。球が想像より大きく見えた（radius="1.25" は直径2.5で、箱より大きい）。
  • Q2: rotation="-90 0 0" を消すと、平面が垂直に立ち上がって「壁」のように見えた。rotation で X軸方向に -90度回転させることで水平な「床」にしていたことがわかった。
  • Q3: z=-3 の箱と円柱が手前に、z=-5 の球が奥に見えた。z の値がマイナスで大きいほど奥に配置される。
