説明

第9回では Socket.IO でテキストチャットを実装しました。
今回はA-Frame の3D空間と Socket.IO のリアルタイム通信を組み合わせ、複数ユーザが同じ3D空間を共有し、互いのアバターが見える仕組みを作ります。

1.1 A-Frame + Socket.IO の組み合わせ 課題に出ます

オンラインゲームでは、自分のキャラクターを動かすと、他のプレイヤーの画面でもそのキャラクターが動きます。
今回作るのは、まさにその仕組みのWeb版です。ブラウザで3D空間に入り、他の人のアバターがリアルタイムに動くのが見えます。

第4〜8回で学んだ A-Frame（3D表示）と、第9回で学んだ Socket.IO（リアルタイム通信）を組み合わせます。

技術役割この回での使い方
Node.js + ExpressサーバA-FrameのHTMLファイルを配信し、Socket.IO で接続を管理する
Socket.IOリアルタイム通信各ユーザのカメラ位置・回転データを他の全ユーザに送る
A-Frame3D表示受信したデータをもとに、他ユーザのアバターを3D空間に表示・移動する

1.2 アバターの概念 課題に出ます

ビデオ会議ではカメラの映像で相手の存在がわかります。3D空間では映像の代わりにアバター（箱や人型の3Dモデル）で「そこに誰かがいる」ことを表現します。

アバターとは、3D空間内で各ユーザを表すエンティティです。今回は色付きの箱（a-box）をアバターとして使います。

  • サーバに接続したユーザ1人につき、1つのアバター（a-box）が3D空間に生成される
  • 自分のアバターは表示しない（自分のカメラ視点で見ているため）
  • 他のユーザが移動すると、そのアバターの位置・回転がリアルタイムに更新される
  • ユーザが切断すると、そのアバターは3D空間から削除される

1.3 位置・回転データの送信 課題に出ます

A-Frame のカメラ（ユーザの視点）の position（位置: x, y, z）と rotation（回転: x, y, z）を定期的にサーバに送信します。

データ内容例
position3D空間内のカメラ座標{x: 0, y: 1.6, z: 3}
rotationカメラの向き（度数）{x: 0, y: -45, z: 0}

これらのデータを 一定間隔（例: 100ms ごと）で socket.emit('move', data) してサーバに送り、サーバが全クライアントに配信します。

なぜ定期送信なのか？ 課題に出ます

マウスやキーボードの入力イベントは非常に高頻度で発生します。すべてのイベントで送信すると、通信量が膨大になりサーバに負荷がかかります。setInterval で一定間隔に間引くことで、滑らかさと通信効率のバランスを取ります。

1.4 他ユーザのアバター生成と更新

他のユーザから位置データを受信したとき、そのユーザのアバターがまだ存在しなければ新規生成し、存在すれば位置を更新します。

JavaScript（概念コード）
// 他ユーザの位置データを受信したとき
socket.on('moved', function(data) {
  var avatar = document.getElementById('avatar-' + data.id);
  if (!avatar) {
    // アバターがなければ新規作成
    avatar = document.createElement('a-box');
    avatar.setAttribute('id', 'avatar-' + data.id);
    avatar.setAttribute('color', data.color);
    scene.appendChild(avatar);
  }
  // 位置と回転を更新
  avatar.setAttribute('position', data.position);
  avatar.setAttribute('rotation', data.rotation);
});

1.5 ユーザID管理 ── socket.id 課題に出ます

Socket.IO では、接続するたびに各クライアントに一意の ID（socket.id）が自動で割り当てられます。

  • socket.id は "xK7-2abc..." のようなランダム文字列
  • この ID を使って「誰の位置データか」「誰のアバターか」を識別する
  • サーバ側でユーザ一覧を管理し、新規接続・切断を追跡する

1.6 ユーザ離脱時のクリーンアップ 課題に出ます

ビデオ会議で誰かが退室すると、その人の映像枠が消えます。3D空間でも同じように、ユーザが切断したらそのアバターを削除しないと、動かない幽霊アバターが残り続けます。

サーバの 'disconnect' イベントで、離脱したユーザの ID を全クライアントに通知し、各クライアントが該当のアバターを DOM から削除します。

イベントサーバの処理クライアントの処理
connectionユーザを一覧に追加、色を割り当て（接続完了）
move受信した位置データを他の全員に転送受信したデータでアバターを生成/更新
disconnectユーザを一覧から削除、全員に通知該当アバターをシーンから削除

📖 キーワード

アバター3D空間内で各ユーザを表すエンティティ。ここでは色付きの箱（a-box）を使用。他ユーザの存在と位置を視覚的に示す。
setInterval一定間隔でコールバック関数を実行するJavaScript関数。ここでは100msごとにカメラ位置をサーバに送信する用途で使う。
position / rotation3D空間内のオブジェクトの位置(x,y,z)と回転角度(x,y,z)。カメラのこれらの値を送受信してアバターを同期する。
クリーンアップユーザ切断時にアバターをDOMから削除する処理。これを怠ると動かない「幽霊アバター」が残り続ける。

アバター同期のデータフロー

ユーザAが3D空間内で移動したとき、ユーザBの画面でアバターが動くまでの流れを追います。

手順場所処理内容
1ユーザA（ブラウザ）WASDキーで移動 → カメラの position/rotation が変わる

5ユーザB（A-Frame）ユーザAのアバター（a-box）が3D空間内で移動して見える

A-Frameページを配信し、位置データをログに表示する

まずはシンプルな構成で、クライアントのカメラ位置をサーバのコンソールに表示します。

ファイル1: server.js

JavaScript ── server.js
var express = require('express');
var http = require('http');
var { Server } = require('socket.io');

var app = express();
var server = http.createServer(app);
var io = new Server(server);

app.use(express.static('public'));

io.on('connection', function(socket) {
  console.log('接続:', socket.id);

  socket.on('move', function(data) {
    console.log(socket.id, '位置:', data.position);
  });

  socket.on('disconnect', function() {
    console.log('切断:', socket.id);
  });
});

server.listen(3000, function() {
  console.log('http://localhost:3000 でサーバ起動');
});

コード体験 2

フルアバター同期 ── 他ユーザが箱として見える

サーバとクライアントを改修し、他のユーザが色付きの箱として3D空間に表示されるようにします。

ファイル1: server.js を以下で上書き

JavaScript ── server.js
var express = require('express');
var http = require('http');
var { Server } = require('socket.io');

var app = express();
var server = http.createServer(app);
var io = new Server(server);

app.use(express.static('public'));

// ユーザ管理用オブジェクト
var users = {};
var colors = ['#FF5252', '#FFB800', '#4FC3F7', '#76B900', '#E040FB', '#FF6E40'];

io.on('connection', function(socket) {
  // 新しいユーザに色を割り当てる
  var color = colors[Object.keys(users).length % colors.length];
  users[socket.id] = { color: color };
  console.log('接続:', socket.id, '色:', color);

  // 接続したユーザに自分の色を通知
  socket.emit('myColor', { color: color });

  // 位置データを受信して他の全員に転送
  socket.on('move', function(data) {
    socket.broadcast.emit('moved', {
      id: socket.id,
      position: data.position,
      rotation: data.rotation,
      color: users[socket.id].color
    });
  });

  // 切断時: 全員に通知してアバター削除を依頼
  socket.on('disconnect', function() {
    console.log('切断:', socket.id);
    delete users[socket.id];
    io.emit('userLeft', { id: socket.id });
  });
});

server.listen(3000, function() {
  console.log('http://localhost:3000 でサーバ起動');
});

ファイル2: public/index.html を以下で上書き

HTML ── public/index.html
<!DOCTYPE html>
<html lang="ja">
<head>

  <title>Avatar Sync</title>
  <script src="https://aframe.io/releases/1.7.1/aframe.min.js"></script>

</head>
<body>
  <a-scene>
    <!-- 床 -->
    <a-plane position="0 0 0" rotation="-90 0 0"
            width="30" height="30" color="#7BC8A4"></a-plane>

    <!-- 目印のオブジェクト -->
    <a-box position="0 0.5 -8" color="#4CC3D9" width="2" height="2" depth="2"></a-box>
    <a-cylinder position="5 1.5 -5" radius="0.3" height="3" color="#FFC65D"></a-cylinder>
    <a-sphere position="-5 1 -5" radius="1" color="#EF2D5E"></a-sphere>

    <!-- 空 -->
    <a-sky color="#87CEEB"></a-sky>
  </a-scene>

  <!-- 接続中ユーザ数の表示 -->
  <div id="user-count" style="position:fixed;top:10px;left:10px;color:white;background:rgba(0,0,0,0.6);padding:8px 16px;border-radius:8px;font-family:sans-serif;font-size:14px;z-index:999">
    接続中: 1人
  </div>

  <script>
    var socket = io();
    var scene = document.querySelector('a-scene');
    var camera = document.querySelector('[camera]');
    var userCount = 1;

    // 100msごとにカメラ位置を送信
    setInterval(function() {
      if (!camera) return;
      var pos = camera.getAttribute('position');
      var rot = camera.getAttribute('rotation');
      socket.emit('move', {
        position: { x: pos.x, y: pos.y, z: pos.z },
        rotation: { x: rot.x, y: rot.y, z: rot.z }
      });
    }, 100);

    // 他ユーザの位置データを受信 → アバター生成/更新
    socket.on('moved', function(data) {
      var avatarId = 'avatar-' + data.id;
      var avatar = document.getElementById(avatarId);

      if (!avatar) {
        // 新しいアバターを生成
        avatar = document.createElement('a-box');
        avatar.setAttribute('id', avatarId);
        avatar.setAttribute('width', '0.5');
        avatar.setAttribute('height', '0.5');
        avatar.setAttribute('depth', '0.5');
        avatar.setAttribute('color', data.color);
        scene.appendChild(avatar);
        userCount++;
        document.getElementById('user-count').textContent = '接続中: ' + userCount + '人';
      }

      // 位置と回転を更新
      avatar.setAttribute('position', data.position);
      avatar.setAttribute('rotation', {
        x: 0,
        y: data.rotation.y,
        z: 0
      });
    });

    // ユーザ離脱 → アバター削除
    socket.on('userLeft', function(data) {
      var avatar = document.getElementById('avatar-' + data.id);
      if (avatar) {
        avatar.parentNode.removeChild(avatar);
        userCount--;
        document.getElementById('user-count').textContent = '接続中: ' + userCount + '人';
      }
    });
  </script>
</body>
</html>
