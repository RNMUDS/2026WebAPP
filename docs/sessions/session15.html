発表の進め方

1人（またはグループ）あたり 3〜5分 で、制作したマルチユーザ3D Webアプリケーションを発表します。

発表の流れ

1. テーマ紹介 → 2. デモ実演 → 3. 技術説明 → 4. 質疑応答

発表内容（3〜5分）

パート内容目安
テーマ紹介 どんな3D空間を作ったか。テーマの着想やコンセプトを簡潔に説明する。 30秒
デモ実演 サーバを起動し、ブラウザで3D空間を見せる。複数タブで接続し、アバター同期とチャット機能を実演する。 2分
技術説明 使用した技術、工夫した点、苦労した点を説明する。コードの一部を見せてもよい。 1〜2分
質疑応答 聴衆や他の発表者からの質問に答える。 30秒

発表時のポイント

  • デモの事前準備: 発表前に node server.js でサーバを起動しておく
  • タブを2つ開いておく: マルチユーザ機能を見せるために、事前にブラウザタブを2つ用意する
  • うまくいかない場合: スクリーンショットやコードを見せて説明する（動かなくても評価対象）
  • 聴衆に向けて: 画面だけでなく、聴衆に向かって話す

評価基準

評価項目内容配点
完成度必須機能（3Dシーン、アバター同期、チャット）が動作するか30%
技術的な工夫オプション機能、独自の工夫、コードの品質25%
創造性テーマの独自性、3D空間のデザイン、ユーザ体験20%
発表の質説明のわかりやすさ、デモの見せ方、質疑への対応15%
コード・ドキュメントソースコード一式、README.md の質10%

今後の学びへ

この授業で学んだ技術は、さらに深く学ぶことでより高度なWebアプリケーションを作れるようになります。
興味のある分野を見つけて、ぜひ自主的に学び続けてください。

A

A-Frame 公式ドキュメント

A-Frameの全コンポーネント・機能の公式リファレンス。今回の授業で扱わなかったパーティクルシステム、物理演算、VRコントローラ対応なども学べます。

https://aframe.io/docs/1.7.0/introduction/
S

Socket.IO 公式ドキュメント

Socket.IOの高度な機能（ルーム、名前空間、認証、スケーリング）を学べます。より大規模なリアルタイムアプリケーション構築に必要な知識です。

https://socket.io/docs/v4/
T

Three.js

A-Frameの内部で使われている3Dライブラリです。A-Frameより低レベルですが、自由度が高く、より高度な3D表現が可能です。ゲーム開発やインタラクティブな3Dコンテンツ制作に向いています。

https://threejs.org/
W

WebXR Device API

ブラウザからVR/ARデバイスにアクセスするためのWeb標準API。Meta Quest、Apple Vision Proなどのデバイスで動くイマーシブな体験を作れます。

https://developer.mozilla.org/ja/docs/Web/API/WebXR_Device_API
N

Node.js + Express で本格的なWebアプリ

認証、データベース（MongoDB, PostgreSQL）、APIセキュリティ、デプロイ（Heroku, Vercel, AWS）など、本格的なWebアプリケーション開発に必要な技術を学べます。

https://expressjs.com/
P

ポートフォリオとして活用する

この授業で制作したアプリケーションは、就職活動や大学院進学のポートフォリオとして活用できます。GitHubにソースコードを公開し、READMEにデモ動画やスクリーンショットを追加してみてください。
