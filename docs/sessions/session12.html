説明

前回までにSocket.IOでリアルタイム通信を実装しました。
しかし、サーバを再起動するとすべてのデータが消えてしまう問題があります。今回はデータベース（SQLite）を導入し、データを永続化する方法を学びます。

1.1 なぜデータベースが必要か 課題に出ます

メモ帳に書いた内容は、メモ帳を閉じると消えます。でもノートに書いておけば、次に開いたときもそのまま残っています。
サーバのメモリは「メモ帳」、データベースは「ノート」のようなものです。

これまでのチャットアプリやアバター同期では、サーバのメモリ（変数）にデータを保持していました。

保存方法特徴サーバ再起動時
メモリ（変数）高速だがプロセス終了で消えるすべてのデータが消失
データベースファイルに永続保存されるデータが残る

具体的な問題:

  • チャットの履歴が再起動で全部消える
  • 3Dオブジェクトの位置や色の変更が保存されない
  • 新しく接続したユーザに過去のデータを送れない

データベースを使えば、これらの問題をすべて解決できます。

1.2 SQLite とは 課題に出ます

大きな図書館（MySQL, PostgreSQL）は管理人が常駐し、たくさんの人が同時に使えますが、設置が大変です。
SQLite は「自分だけの小さな本棚」のようなもので、設置不要ですぐに使えます。

SQLite は、ファイルベースの軽量なリレーショナルデータベースです。

特徴説明
サーバ不要別途データベースサーバを起動する必要がない（組み込み型）
ファイルベースデータベース全体が1つのファイル（例: chat.db）に保存される
軽量インストールが簡単で、小〜中規模のアプリに最適
SQL対応標準的なSQL文でデータを操作できる

SQLite vs MySQL / PostgreSQL

MySQL や PostgreSQL は大規模なサービス向けですが、この授業で作る規模のアプリなら SQLite で十分です。設定もインストールも簡単で、学習コストが低いのが利点です。

1.3 better-sqlite3 パッケージ

Node.js から SQLite を操作するには better-sqlite3 というパッケージを使います。

Terminal
npm install better-sqlite3

基本的な使い方:

JavaScript
// better-sqlite3 を読み込む
const Database = require('better-sqlite3');

// データベースファイルを開く（なければ自動作成）
const db = new Database('myapp.db');

これだけで myapp.db というファイルが作成され、データベースとして使えるようになります。

1.4 基本的なSQL文 課題に出ます

SQL（Structured Query Language）は、データベースを操作するための言語です。以下の4つが基本です。

SQL文意味例
CREATE TABLEテーブル（表）を作成するCREATE TABLE messages (id INTEGER PRIMARY KEY, text TEXT)
INSERT INTOデータを追加するINSERT INTO messages (text) VALUES ('こんにちは')
SELECTデータを取得するSELECT * FROM messages
UPDATEデータを更新するUPDATE messages SET text = '修正' WHERE id = 1

テーブルとは 課題に出ます

テーブルはExcelのシートのようなものです。列（カラム）がデータの種類、行（レコード）が個々のデータを表します。

idnametextcreated_at
1太郎こんにちは2026-07-15 16:30:00
2花子やあ！2026-07-15 16:30:15
3太郎元気？2026-07-15 16:30:30

better-sqlite3 での書き方:

JavaScript
// テーブルを作成
db.exec(`
  CREATE TABLE IF NOT EXISTS messages (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    text TEXT NOT NULL,
    created_at TEXT DEFAULT (datetime('now', 'localtime'))
  )
`);

// データを追加（プリペアドステートメント）
const insert = db.prepare('INSERT INTO messages (name, text) VALUES (?, ?)');
insert.run('太郎', 'こんにちは');

// データを全件取得
const rows = db.prepare('SELECT * FROM messages').all();
console.log(rows);
// → [{ id: 1, name: '太郎', text: 'こんにちは', created_at: '2026-07-15 16:30:00' }]

プリペアドステートメントとは？

? をプレースホルダとして使い、後から値を渡す方法です。SQL文に直接値を埋め込むよりも安全（SQLインジェクション対策）で高速です。

1.5 オブジェクト状態の永続化

3D空間内のオブジェクト（位置、色など）をデータベースに保存すれば、サーバを再起動しても状態が維持されます。

JavaScript
// オブジェクト状態テーブル
db.exec(`
  CREATE TABLE IF NOT EXISTS objects (
    obj_id TEXT PRIMARY KEY,
    x REAL DEFAULT 0,
    y REAL DEFAULT 0,
    z REAL DEFAULT 0,
    color TEXT DEFAULT '#FFFFFF'
  )
`);

// オブジェクトの位置を保存（なければ追加、あれば更新）
const upsert = db.prepare(`
  INSERT INTO objects (obj_id, x, y, z, color)
  VALUES (?, ?, ?, ?, ?)
  ON CONFLICT(obj_id) DO UPDATE SET x=?, y=?, z=?, color=?
`);

function saveObject(id, x, y, z, color) {
  upsert.run(id, x, y, z, color, x, y, z, color);
}

1.6 チャット履歴の保存

チャットメッセージをデータベースに保存すれば、サーバ再起動後もメッセージが残り、新規接続ユーザにも過去のメッセージを表示できます。

  • メッセージ送信時に INSERT INTO でデータベースに保存
  • 新規接続時に SELECT で過去メッセージを取得して送信
  • メッセージ数が多い場合は ORDER BY ... LIMIT で最新N件だけ取得

1.7 新規接続時のデータ復元 課題に出ます

新しいユーザが接続したとき、データベースからデータを読み込んで送信します。

JavaScript
io.on('connection', (socket) => {
  // 1. 保存済みのオブジェクト状態を送信
  const objects = db.prepare('SELECT * FROM objects').all();
  socket.emit('init-objects', objects);

  // 2. 過去のチャット履歴（最新50件）を送信
  const messages = db.prepare(
    'SELECT * FROM messages ORDER BY id DESC LIMIT 50'
  ).all().reverse();
  socket.emit('chat-history', messages);
});

これにより、途中から参加したユーザも過去の状態を確認できるようになります。

チャットアプリにメッセージ永続化を追加する

同じ Web12 フォルダに以下の2ファイルを作成してください。

  1. ターミナルで npm install express socket.io を実行（未インストールの場合）

JavaScript ── server.js
const express = require('express');
const http = require('http');
const { Server } = require('socket.io');
const Database = require('better-sqlite3');

// --- Express + Socket.IO セットアップ ---
const app = express();
const server = http.createServer(app);
const io = new Server(server);

app.use(express.static('.'));

// --- データベースセットアップ ---
const db = new Database('chat.db');

db.exec(`
  CREATE TABLE IF NOT EXISTS messages (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    text TEXT NOT NULL,
    created_at TEXT DEFAULT (datetime('now', 'localtime'))
  )
`);

// プリペアドステートメントを事前に用意
const insertMsg = db.prepare(
  'INSERT INTO messages (name, text) VALUES (?, ?)'
);
const getHistory = db.prepare(
  'SELECT * FROM messages ORDER BY id DESC LIMIT 50'
);

// --- Socket.IO イベント ---
io.on('connection', (socket) => {
  console.log('接続:', socket.id);

  // 接続時に過去のチャット履歴を送信
  const history = getHistory.all().reverse();
  socket.emit('chat-history', history);

  // メッセージ受信 → DB保存 → 全員に配信
  socket.on('chat-message', (data) => {
    // データベースに保存
    const result = insertMsg.run(data.name, data.text);

    // 保存したメッセージ（IDと日時付き）を全員に配信
    const saved = {
      id: result.lastInsertRowid,
      name: data.name,
      text: data.text,
      created_at: new Date().toLocaleString('ja-JP')
    };
    io.emit('chat-message', saved);
  });

  socket.on('disconnect', () => {
    console.log('切断:', socket.id);
  });
});

// --- サーバ起動 ---
server.listen(3000, () => {
  console.log('サーバ起動: http://localhost:3000');
});

    // 過去の履歴を受信して表示

      const name = document.getElementById('name').value || 'ゲスト';
      const text = document.getElementById('text').value;
      if (!text) return;
      socket.emit('chat-message', { name, text });
      document.getElementById('text').value = '';
    }

      if (e.key === 'Enter') send();

標準 標準課題 1 解答（データベースとSQLの基礎知識）

問1.

SQL文意味使う場面
CREATE TABLE新しいテーブル（表）を作成するアプリの初回起動時にデータの入れ物を用意する
INSERT INTOテーブルに新しいデータ（行）を追加するチャットメッセージの送信時、ユーザ登録時など
SELECTテーブルからデータを取得するチャット履歴の表示、ユーザ情報の確認など
UPDATE既存のデータを変更するオブジェクトの色や位置の変更、プロフィール更新など

問2.

観点メモリ（変数）データベース
サーバ再起動時すべて消失するデータが残る（永続化される）
アクセス速度非常に高速（ナノ秒単位）メモリより遅い（ファイルI/Oが発生するため）
データ量の限界サーバのメモリ容量に制限される（数GB程度）ディスク容量まで保存可能（数百GB以上）

問3.

場面DB or メモリ？理由
Aユーザの登録情報DB永続的に保持する必要があるデータ。サーバ再起動で消えてはならない。
B現在接続中のユーザ一覧メモリ接続中だけ必要な一時的なデータ。切断されたら不要になる。
Cチャットの過去ログDB後から見返す必要がある履歴データ。永続化が必要。
Dマウスカーソルの現在位置メモリ常に変化する一時データ。毎フレーム更新されるため、DBに保存すると遅すぎる。
改造 標準課題 2 解答例（コード改造） 
const express = require('express');
const http = require('http');
const { Server } = require('socket.io');
const Database = require('better-sqlite3');

const app = express();
const server = http.createServer(app);
const io = new Server(server);

app.use(express.static('.'));

const db = new Database('chat.db');

// 改造1: room 列を追加
db.exec(`
  CREATE TABLE IF NOT EXISTS messages (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    text TEXT NOT NULL,
    room TEXT DEFAULT 'general',
    created_at TEXT DEFAULT (datetime('now', 'localtime'))
  )
`);

// 改造2: INSERT に room を追加
const insertMsg = db.prepare(
  'INSERT INTO messages (name, text, room) VALUES (?, ?, ?)'
);
// 改造3: LIMIT を 100 に変更
const getHistory = db.prepare(
  'SELECT * FROM messages ORDER BY id DESC LIMIT 100'
);

// 改造4: メッセージ数を表示
const msgCount = db.prepare('SELECT COUNT(*) as count FROM messages').get().count;
console.log('現在のメッセージ数: ' + msgCount + '件');

io.on('connection', (socket) => {
  console.log('接続:', socket.id);

  const history = getHistory.all().reverse();
  socket.emit('chat-history', history);

  socket.on('chat-message', (data) => {
    // 改造2: room も保存
    const result = insertMsg.run(data.name, data.text, 'general');

    const saved = {
      id: result.lastInsertRowid,
      name: data.name,
      text: data.text,
      created_at: new Date().toLocaleString('ja-JP')
    };
    io.emit('chat-message', saved);
  });

  socket.on('disconnect', () => {
    console.log('切断:', socket.id);
  });
});

server.listen(3000, () => {
  console.log('サーバ起動: http://localhost:3000');
});

各改造のポイント:

  • 改造1: CREATE TABLE に room TEXT DEFAULT 'general' を追加。将来的に複数ルームに対応するためのカラム。
  • 改造2: INSERT 文を (name, text, room) に変更し、run に 'general' を追加。
  • 改造3: LIMIT 50 を LIMIT 100 に変更。より多くの履歴を表示できるようになる。
  • 改造4: SELECT COUNT(*) as count FROM messages で件数を取得し、console.log で表示。サーバ起動時に現在の蓄積データ量を確認できる。

server.js:

const express = require('express');
const http = require('http');
const { Server } = require('socket.io');
const Database = require('better-sqlite3');

const app = express();
const server = http.createServer(app);
const io = new Server(server);

app.use(express.static('.'));

const db = new Database('scene.db');

// オブジェクト状態テーブル
db.exec(`
  CREATE TABLE IF NOT EXISTS objects (
    obj_id TEXT PRIMARY KEY,
    color TEXT DEFAULT '#FFFFFF'
  )
`);

// 初期データ（存在しない場合のみ追加）
const insertOrIgnore = db.prepare(
  'INSERT OR IGNORE INTO objects (obj_id, color) VALUES (?, ?)'
);
insertOrIgnore.run('box1', '#4CC3D9');
insertOrIgnore.run('sphere1', '#EF2D5E');
insertOrIgnore.run('cylinder1', '#FFC65D');

const updateColor = db.prepare(
  'UPDATE objects SET color = ? WHERE obj_id = ?'
);
const getAllObjects = db.prepare('SELECT * FROM objects');

const COLORS = ['#FF5252','#4FC3F7','#76B900','#FFB800','#E040FB','#FF6D00','#00BCD4'];

io.on('connection', (socket) => {
  // 接続時に保存済みの色を送信
  const objects = getAllObjects.all();
  socket.emit('init-objects', objects);

  // 色変更リクエスト
  socket.on('change-color', (data) => {
    const newColor = COLORS[Math.floor(Math.random() * COLORS.length)];
    updateColor.run(newColor, data.obj_id);
    io.emit('color-changed', { obj_id: data.obj_id, color: newColor });
  });
});

server.listen(3000, () => {
  console.log('サーバ起動: http://localhost:3000');
});
