説明

前回は ECS アーキテクチャと座標系を学びました。
今回はmaterial コンポーネントを深掘りし、色・質感・テクスチャ（画像）を使ってオブジェクトの見た目を自在に制御する方法を学びます。

1.1 カラー指定方法 課題に出ます

絵の具は「赤」「青」と名前で選べますが、デジタルでは色を数値で正確に指定できます。#FF0000 は「赤255、緑0、青0」で純粋な赤を表します。

A-Frame では以下の3つの方法で色を指定できます。

指定方法書き方の例説明
色名color: red, color: skyblueCSS で定義された色名（約140色）をそのまま使える
16進数（hex）color: #FF6347, color: #333#RRGGBB の形式。各2桁が赤・緑・青の強さ（00〜FF）を表す
RGBcolor: rgb(255, 99, 71)赤・緑・青の値を 0〜255 で直接指定する

16進数カラーの読み方 課題に出ます

#FF6347 を分解すると: FF（赤=255）63（緑=99）47（青=71）→ 赤が強くオレンジ寄りの色（トマト色）。#000000 は黒、#FFFFFF は白、#FF0000 は純粋な赤です。

1.2 material コンポーネント 課題に出ます

同じ形のコップでも、ガラス製なら透明でツヤがあり、陶器製ならマットな質感、金属製なら光を反射します。material コンポーネントはこのような「素材の質感」をコントロールします。

material コンポーネントは、オブジェクトの表面の見た目を制御します。前回までは color だけを使っていましたが、他にも多くのプロパティがあります。

プロパティ値の範囲デフォルト効果
color色名 / hex / rgb#FFF表面の色を指定する
opacity0.0 〜 1.01.0不透明度。0で完全に透明、1で完全に不透明
transparenttrue / falsefalseopacity を使うときは true に設定する必要がある
metalness0.0 〜 1.00.0金属感。1.0で完全な金属表面（周囲を映し込む）
roughness0.0 〜 1.00.5表面の粗さ。0でツルツル（鏡面）、1でザラザラ（マット）
sidefront / back / doublefrontどちら側の面を表示するか
src画像URL / セレクタなしテクスチャ画像を貼り付ける
HTML ── 使用例
<!-- 半透明の青い箱 -->
<a-entity
  geometry="primitive: box"
  material="color: blue; opacity: 0.5; transparent: true"
  position="0 1 -3"></a-entity>

<!-- 金属的な球 -->
<a-entity
  geometry="primitive: sphere"
  material="color: #C0C0C0; metalness: 0.9; roughness: 0.1"
  position="2 1 -3"></a-entity>

1.3 テクスチャ ── 画像をオブジェクトの表面に貼る 課題に出ます

プレゼントの箱に包装紙を巻くように、3Dオブジェクトの表面に画像を貼ることができます。これをテクスチャマッピングと呼びます。

material の src プロパティに画像を指定すると、その画像がオブジェクトの表面に貼り付けられます。

HTML ── 画像URLを直接指定
<a-entity
  geometry="primitive: box"
  material="src: url(brick.jpg)"
  position="0 1 -3"></a-entity>

ただし、この書き方では画像を毎回ダウンロードしてしまいます。複数のオブジェクトで同じ画像を使う場合、次に説明する <a-assets> を使いましょう。

1.4 <a-assets> によるアセット管理 課題に出ます

料理を始める前に食材をまとめて準備する「下ごしらえ」と同じです。<a-assets> で画像を先に読み込んでおけば、3Dシーンの表示がスムーズになります。

<a-assets> は A-Frame のアセット管理システムです。画像・音声・動画などを事前に読み込み（プリロード）してから3Dシーンを描画します。

HTML ── a-assets の使い方
<a-scene>
  <!-- アセットの事前読み込み -->
  <a-assets>
    <img id="brick-tex" src="brick.jpg">
    <img id="wood-tex" src="wood.jpg">
  </a-assets>

  <!-- セレクタ（#id）で参照する -->
  <a-entity
    geometry="primitive: box"
    material="src: #brick-tex"
    position="-2 1 -3"></a-entity>

  <a-entity
    geometry="primitive: plane; width: 4; height: 4"
    material="src: #wood-tex"
    position="0 0 -3"
    rotation="-90 0 0"></a-entity>
</a-scene>
  • <a-assets> は <a-scene> の直下に書く
  • <img id="..."> で画像に ID を付ける
  • material="src: #ID名" で参照する（# を付けるのを忘れない）
  • 同じ画像を複数のオブジェクトで使い回せる
  • 画像ファイルはHTMLと同じフォルダに置くか、パスを指定する

1.5 透明度（opacity）課題に出ます

オブジェクトを半透明にするには、opacity と transparent の両方を設定する必要があります。

HTML
<!-- 50% 透明な赤い球 -->
<a-entity
  geometry="primitive: sphere"
  material="color: red; opacity: 0.5; transparent: true"
  position="0 1 -3"></a-entity>

よくあるミス

opacity: 0.5 だけ書いて transparent: true を付け忘れると、透明にならないことがあります。透明効果を使うときは必ず transparent: true も一緒に書いてください。

1.6 両面表示（side: double）

紙を1枚持って正面から見ると面が見えますが、裏返すと見えなくなります。3Dでも同じで、デフォルトでは表面だけが描画されます。壁や看板の裏側も見せたいときは side: double を使います。

side プロパティは描画する面を制御します。

値効果使いどころ
front表面のみ描画（デフォルト）通常のオブジェクト
back裏面のみ描画内側から見る場合（トンネルの中など）
double両面を描画壁、看板など裏からも見えるもの

前回の発展課題で壁を作ったとき、裏側から見ると壁が消えたかもしれません。side: double を付けると解決します。

例題

metalness と roughness の組み合わせ効果

以下の4つの球は、色は同じですが metalness と roughness が異なります。見た目の違いを確認しましょう。

球metalnessroughness見た目
A0.00.5プラスチックのような標準的な見た目
B0.00.0ツルツルのプラスチック（光沢あり）
C1.00.5少し曇った金属（ブラッシュド加工）
D1.00.0鏡面のような完全な金属（周囲が映る）
  • metalness が高いと金属的に周囲の環境を映し込む
  • roughness が低いとツルツル（光が集中して反射）、高いとザラザラ（光が拡散）
  • この2つの組み合わせで「プラスチック」「ガラス」「金属」「ゴム」などさまざまな質感を表現できる

コード体験 2

テクスチャとアセット管理 ── texture.html

同じ Web05 フォルダにファイルと画像を準備します。

画像の準備

テクスチャ用の画像ファイルが必要です。以下のいずれかの方法で準備してください。

  • 方法A: フリー素材サイト（textures.com や polyhaven.com）からレンガ・木目などの画像をダウンロードし、Web05 フォルダに保存
  • 方法B: 自分で撮った写真や、手持ちの画像ファイル（JPG/PNG）を Web05 フォルダにコピー

ここでは brick.jpg（レンガ）、wood.jpg（木目）という名前で用意した前提で進めます。自分の画像のファイル名が異なる場合は、コード内のファイル名を書き換えてください。

  1. VS Code 左側のエクスプローラで 「新しいファイル」アイコン をクリック
  2. ファイル名を texture.html と入力して Enter
  3. 下のコードを「コピー」→ 貼り付け → 保存

<html lang="ja">
<head>
  <meta charset="UTF-8">

</head>
<body>
  <a-scene>
    <!-- アセットの事前読み込み -->
    <a-assets>
      <img id="brick-tex" src="brick.jpg">

テクスチャを使ったギャラリーを作る

VS Code で Web05 フォルダに gallery.html を新規作成し（エクスプローラの「新しいファイル」アイコン → ファイル名入力）、テクスチャとマテリアルプロパティを活用した展示ギャラリー（美術館風の部屋）を作ってください。

必須要件

  • テクスチャ画像を3枚以上使用する（壁・床・展示物など）
  • <a-assets> で画像をプリロードする
  • metalness / roughness を使って異なる質感のオブジェクトを2つ以上配置する
  • 透明度（opacity + transparent）を1か所以上使う（ガラスケース、窓など）
  • side: double を壁の少なくとも1面に適用する
  • ギャラリーの構成: 床・壁・展示台や額縁を含む空間を作る

提出要件（2点セット）

提出物内容
1. コード gallery.html のコード全体をNotionノートに貼り付ける
2. 音声説明 以下の内容を1〜2分の音声で録音し、Notionノートに添付する
スマホの録音アプリやPCの機能で録音 → Notionに /audio で添付

音声で説明すること:

  • どんなギャラリー/展示空間をイメージしたか
  • どのテクスチャ画像をどこに使ったか
  • metalness/roughness をどのオブジェクトに使い、どんな質感を表現したか
  • 透明度をどこで使い、どんな効果を狙ったか
ヒント: ギャラリーの構成例
  • 壁を3面設置し、各壁にテクスチャ（レンガ・コンクリートなど）を貼る
  • 壁に「額縁」（薄い box + 画像テクスチャ）を掛ける → 絵画の展示
  • 中央にガラスケース（半透明の box）を置き、中に金属的なオブジェクト（metalness 高め）を入れる
  • 床には木目テクスチャ、天井は白い plane

A球シルバー (#C0C0C0)metalness: 0.9 で金属的に周囲を映し込む。roughness: 0.1 でほぼ鏡面。光沢のある金属球。

エンティティ参照している src対応する画像ファイル名何に見えそうか
E1#tex-brickbrick.jpgレンガの壁（幅4m、高さ2mの箱にレンガテクスチャ）

体験 コード体験 1 解答（記録例）
  • Q1: 予測とおおむね合っていた。金属球（C, D）は予想以上に暗く見えた。これは metalness が高いと周囲の環境を映し込むが、シンプルなシーンでは映し込む明るい要素が少ないため。
  • Q2: A（metalness:0, roughness:0.5）は普通のプラスチック風。B（metalness:0, roughness:0）は光沢のあるツルツルした球。C（metalness:1, roughness:0.5）は少し曇った金属で暗め。D（metalness:1, roughness:0）は鏡面のように周囲を映す金属。metalness は金属感、roughness は反射の鮮明さを制御している。
  • Q3: はい、E の後ろにあるオブジェクトが透けて見えた。opacity: 0.4 なので赤い色がかかった半透明の箱を通して背景が見える。
体験 コード体験 2 解答（記録例）
  • Q1: テクスチャなしの茶色い箱は「のっぺり」した単色に見えるが、テクスチャありの箱は木目の模様が表面に描かれて非常にリアルになった。テクスチャは単色よりもはるかにリアリティが高い。
  • Q2: 半透明のテクスチャ箱は、レンガのテクスチャがうっすら見えつつ、後ろのオブジェクトも透けて見えた。テクスチャと透明度は同時に使える。
  • Q3: <a-assets> を使うと画像が事前にまとめて読み込まれるため、同じ画像を複数回使っても1回しかダウンロードされない。使わない場合、各エンティティが個別に画像を読み込むため、表示が遅くなる可能性がある。
