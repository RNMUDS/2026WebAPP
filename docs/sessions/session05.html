説明

前回は ECS アーキテクチャと座標系を学びました。
今回はmaterial コンポーネントを深掘りし、色・質感・テクスチャ（画像）を使ってオブジェクトの見た目を自在に制御する方法を学びます。

前回の復習: ECS とは？

ECS（Entity-Component-System / エンティティ・コンポーネント・システム）は、A-Frame / Three.js が採用している 3D オブジェクトの設計思想です。

用語読み方役割
Entity（エンティティ） エンティティ 「モノ」の入れ物。それ自体は何の見た目も動作も持たない空の容器（<a-entity>）
Component（コンポーネント） コンポーネント エンティティに付ける「機能パーツ」。geometry（形）、material（見た目）、position（位置）など
System（システム） システム 同じ種類のコンポーネントを一括処理する仕組み（A-Frame が裏で自動運用、初心者は直接書かなくてOK）

言い換えると ──「空の Entityに、必要な Componentを組み合わせて貼り付け、3Dオブジェクトを組み立てる」しくみです。
前回は geometry（形）と position（位置）が中心でしたが、今回は material（見た目）コンポーネントを深掘りします。

💡 構造を一言で: <a-entity geometry="..." material="color: red; metalness: 1" position="0 1 -3"> ── この1行が「Entity + 3つの Component（geometry / material / position）」の組み合わせです。今回は material の中身を細かく調整します。

記法意味
<a-entity geometry="..." material="..."> 明示的に Entity + Component を分けて書く（前回学んだECS記法）
<a-box color="red"> プリミティブ記法（実は内部で「Entity + geometry='primitive:box' + material='color:red'」に変換されている）

つまり前回の <a-box color="red"> は、裏で material コンポーネントが自動的に付いていたのです。今回は material を明示的に書き、その中の color / metalness / roughness / opacity / src を細かく操作することで、同じ箱でも「金属」「ゴム」「ガラス」など多様な見た目に変えられるようになります。

用語の読み方

用語読み方意味（ひとことで）
Three.jsスリージェイエスWebGL を使いやすくする3DライブラリのJavaScript版
WebGLウェブジーエルブラウザで3D描画を行う標準API

1.1 カラー指定方法

絵の具は「赤」「青」と名前で選べますが、デジタルでは色を数値で正確に指定できます。#FF0000 は「赤255、緑0、青0」で純粋な赤を表します。

A-Frame では以下の3つの方法で色を指定できます。

指定方法書き方の例説明
色名color: red, color: skyblueCSS で定義された色名（約140色）をそのまま使える
16進数（hex）color: #FF6347, color: #333#RRGGBB の形式。各2桁が赤・緑・青の強さ（00〜FF）を表す
RGBcolor: rgb(255, 99, 71)赤・緑・青の値を 0〜255 で直接指定する

16進数カラーの読み方

#FF6347 を分解すると: FF（赤=255）63（緑=99）47（青=71）→ 赤が強くオレンジ寄りの色（トマト色）。#000000 は黒、#FFFFFF は白、#FF0000 は純粋な赤です。

🔹 パート4: 色相環パレット（HSL）

色は HSL（色相 H、彩度 S、明度 L）でも考えられます。色相 H を 0°〜360° に変えると、虹のように色がぐるりと一周します。テーマカラーをまとめて設計するときに便利です。

1.2 material コンポーネント

同じ形のコップでも、ガラス製なら透明でツヤがあり、陶器製ならマットな質感、金属製なら光を反射します。material コンポーネントはこのような「素材の質感」をコントロールします。

material コンポーネントは、オブジェクトの表面の見た目を制御します。前回までは color だけを使っていましたが、他にも多くのプロパティがあります。

プロパティ値の範囲デフォルト効果
color色名 / hex / rgb#FFF表面の色を指定する
opacity0.0 〜 1.01.0不透明度。0で完全に透明、1で完全に不透明
transparenttrue / falsefalseopacity を使うときは true に設定する必要がある
metalness0.0 〜 1.00.0金属感。1.0で完全な金属表面（周囲を映し込む）
roughness0.0 〜 1.00.5表面の粗さ。0でツルツル（鏡面）、1でザラザラ（マット）
sidefront / back / doublefrontどちら側の面を表示するか
src画像URL / セレクタなしテクスチャ画像を貼り付ける
HTML ── 使用例
<!-- 半透明の青い箱 -->
<a-entity
  geometry="primitive: box"
  material="color: blue; opacity: 0.5; transparent: true"
  position="0 1 -3"></a-entity>

<!-- 金属的な球 -->
<a-entity
  geometry="primitive: sphere"
  material="color: #C0C0C0; metalness: 0.9; roughness: 0.1"
  position="2 1 -3"></a-entity>

1.3 テクスチャ ── 画像をオブジェクトの表面に貼る

プレゼントの箱に包装紙を巻くように、3Dオブジェクトの表面に画像を貼ることができます。これをテクスチャマッピングと呼びます。

material の src プロパティに画像を指定すると、その画像がオブジェクトの表面に貼り付けられます。

HTML ── 画像URLを直接指定
<a-entity
  geometry="primitive: box"
  material="src: url(brick.jpg)"
  position="0 1 -3"></a-entity>

ただし、この書き方では画像を毎回ダウンロードしてしまいます。複数のオブジェクトで同じ画像を使う場合、次に説明する <a-assets> を使いましょう。

1.4 <a-assets> によるアセット管理

料理を始める前に食材をまとめて準備する「下ごしらえ」と同じです。<a-assets> で画像を先に読み込んでおけば、3Dシーンの表示がスムーズになります。

<a-assets> は A-Frame のアセット管理システムです。画像・音声・動画などを事前に読み込み（プリロード）してから3Dシーンを描画します。

HTML ── a-assets の使い方
<a-scene>
  <!-- アセットの事前読み込み -->
  <a-assets>
    <img id="brick-tex" src="brick.jpg">
    <img id="wood-tex" src="wood.jpg">
  </a-assets>

  <!-- セレクタ（#id）で参照する -->
  <a-entity
    geometry="primitive: box"
    material="src: #brick-tex"
    position="-2 1 -3"></a-entity>

  <a-entity
    geometry="primitive: plane; width: 4; height: 4"
    material="src: #wood-tex"
    position="0 0 -3"
    rotation="-90 0 0"></a-entity>
</a-scene>
  • <a-assets> は <a-scene> の直下に書く
  • <img id="..."> で画像に ID を付ける
  • material="src: #ID名" で参照する（# を付けるのを忘れない）
  • 同じ画像を複数のオブジェクトで使い回せる
  • 画像ファイルはHTMLと同じフォルダに置くか、パスを指定する

1.5 透明度（opacity）

オブジェクトを半透明にするには、opacity と transparent の両方を設定する必要があります。

HTML
<!-- 50% 透明な赤い球 -->
<a-entity
  geometry="primitive: sphere"
  material="color: red; opacity: 0.5; transparent: true"
  position="0 1 -3"></a-entity>

よくあるミス

opacity: 0.5 だけ書いて transparent: true を付け忘れると、透明にならないことがあります。透明効果を使うときは必ず transparent: true も一緒に書いてください。

Q&A: 「opacity を 0.5 にしたのに透明にならない？」

Q: material="opacity: 0.5" と書いたのに、オブジェクトが半透明にならない。これはバグですか？

A: バグではなく仕様です。Three.js（A-Frame の内部エンジン）では、描画速度を上げるためデフォルトで「すべて不透明として扱う」最適化が入っています。transparent: true を明示して初めて、透明用の描画パイプラインに切り替わります。具体的には次の3パターンを覚えてください:

書き方結果
opacity: 0.5 のみ透明にならない（不透明として描画）
transparent: true のみ変化なし（opacity デフォルト 1.0 のまま）
opacity: 0.5; transparent: true正しく半透明になる

1.6 両面表示（side: double）

紙を1枚持って正面から見ると面が見えますが、裏返すと見えなくなります。3Dでも同じで、デフォルトでは表面だけが描画されます。壁や看板の裏側も見せたいときは side: double を使います。

side プロパティは描画する面を制御します。

値効果使いどころ
front表面のみ描画（デフォルト）通常のオブジェクト
back裏面のみ描画内側から見る場合（トンネルの中など）
double両面を描画壁、看板など裏からも見えるもの

前回の発展課題で壁を作ったとき、裏側から見ると壁が消えたかもしれません。side: double を付けると解決します。

1.7 PBR ── 物理ベースレンダリングとは何か

現実の世界では、リンゴの赤や鉄のツヤは「光がどう反射するか」で決まります。PBR は「現実の物理法則どおりに光をシミュレートして、リアルな見た目を作る仕組み」のことです。3D業界の標準になっており、A-Frame の metalness と roughness も PBR の用語です。

PBR（Physically Based Rendering） は「ぶつりベース レンダリング」と読みます。映画 (Pixar/Disney) でも、ゲーム (Unreal/Unity) でも、A-Frame でも、いまや3D描画の業界標準です。覚えるべき2つの軸はとてもシンプルです:

軸意味現実世界の例
metalness金属か / 金属じゃないか の二択1.0=鉄・金・銅 ／ 0.0=木・紙・プラスチック・肌
roughness表面がツルツルか / ザラザラか0.0=新品の鏡・ピアノの黒鍵 ／ 1.0=チョーク・古い木材

覚え方: 物体は 「金属かどうか」 ✕ 「ツルツルか」 の2軸でだいたい表現できる、という発想です。色を決めたあと、この2つのスライダーを動かすだけでリアルな質感が作れます。

1方向 に集中 ② ザラザラ (roughness=1) 光源 表面（微細な凹凸） 拡散反射 → 光が 全方向 に散る ③ 金属 vs 非金属 非金属 metal=0 自分の色 光を吸収・ 一部反射 金属 metal=1 周囲を映す 空・地面を そのまま反射 → 物理的に異なる素材 金属は「自由電子が光を全反射」する のに対し、非金属は「光を吸収・拡散」する

3つの実用ルール:

  • ルール1 ── metalness は 0 か 1 に振り切るのが基本。 中途半端な値（例: 0.5）は「塗装された金属」など特殊な表現用です。
  • ルール2 ── roughness はスライダーのように 0.0〜1.0 を試す。 0=鏡、0.3=磨かれた金属、0.5=普通、0.8=布・ゴム、1.0=チョーク。
  • ルール3 ── 金属（metalness=1）は <a-sky> と光がないと黒くなる。 周囲の何かを映すことで金属らしさが出るためです。

📖 用語の由来

PBR は2010年代に映画業界（Pixar など）が確立し、ゲーム業界（Unreal Engine 4, 2014〜）で広く普及しました。Disney Principled BRDF という有名な論文が現代PBRの基礎です。「同じ素材は、どのレンダラーで描画しても同じに見える」よう、物理量で素材を表現するのが目標です。A-Frame/Three.js もこの規格に従っているため、ここで学ぶ metalness/roughness は Unity でも Unreal でも Blender でも通用します。

Q&A: 「metalness=1.0 にしたのに金属に見えない？」

Q: metalness: 1.0 に設定したのに、シルバーっぽくならずに黒くなってしまいました。なぜ？

A: 金属は「自分自身で発色する」ものではなく「周囲の光や景色を映し返す」素材です。metalness=1.0 のオブジェクトは、空が暗ければ暗く、明るい部屋なら明るく見えます。シーンに <a-sky> や十分な光源がないと、金属球は「映すものが何もない」ので黒く見えます。対処法は次の3つ:

  • <a-sky color="#87CEEB"></a-sky> を入れて、青空を映し込ませる
  • 光源（<a-light type="directional">）を追加して、ハイライトを作る
  • roughness を 0.3〜0.5 にして、反射をぼかして「曇った金属」にする

非金属（metalness=0）なら「自分の色」で光るのに、金属はなぜ違うのか？ それは現実の物理で「金属は自由電子が光をほぼ全反射する」のに対して、非金属は「光を一部吸収して自分の色を返す」性質があるためです。

📖 コラム: マテリアルを20個まとめて作りたい！── JavaScript の for ループで色相環を自動生成

色相環の12〜20色を1個ずつ HTML に書くと、120行以上のコピペになって大変です。
こんなときは JavaScript の for ループで「色相 H を 360°/20 = 18° ずつずらしたマテリアル」を動的に生成すると、わずか数行で済みます。

仕組み

  • for (let i = 0; i < 20; i++) で 0〜19 の連番を回す
  • const hue = i * 18; で色相を 0°, 18°, 36°, ..., 342° と進める
  • document.createElement('a-sphere') で球を作る
  • setAttribute('color', `hsl(${hue}, 80%, 55%)`) で HSL カラーを指定
  • 位置を弧（円周上）に配置: angle = i * (2π/20)、x = cos(angle) * 半径、z = sin(angle) * 半径
🔑 ポイント: HSL の H を 等間隔（360°/個数）でずらすと、全色が美しいグラデーションでつながります。S（彩度）と L（明度）は固定にすることで、明るさ・鮮やかさを揃えたマテリアル一式が作れます。 
JavaScript ── classroom-v2.html の <script> の中身（最小例）
const scene = document.querySelector('a-scene');
const N = 20;          // 20個の球
const radius = 3;     // 半径3mの円周上に配置

for (let i = 0; i < N; i++) {
  const sphere = document.createElement('a-sphere');

  // 色相環: H を 0°→360° に等間隔で変化
  const hue = i * (360 / N);    // 0, 18, 36, ..., 342

  // 円周上の角度（ラジアン）
  const angle = i * (2 * Math.PI / N);
  const x = Math.cos(angle) * radius;
  const z = Math.sin(angle) * radius - 5;  // カメラから5m奥

  sphere.setAttribute('position', `${x} 1.5 ${z}`);
  sphere.setAttribute('radius', '0.4');
  sphere.setAttribute('material',
    `color: hsl(${hue}, 80%, 55%); metalness: 0.5; roughness: 0.4`);

  scene.appendChild(sphere);
}

📺 実行結果イメージ ── 20個の異なる色相の球が弧を描く

💡 発展のヒント: N = 20 を 360 に変えると1° おきの完全な色相環に、roughness も i / N で変化させるとツルツルからマットへの質感グラデーションになります。クリエイティブ課題で「テーマカラーをまとめて配置したい」ときの強力な武器です。

例題 例題1 やってみよう 解答（ゴムボールを表現する metalness / roughness の予測）

ゴムボールは「金属ではない（非金属）」かつ「表面はマットでザラっとしている」素材です。したがって以下の値が妥当な予測になります。

metalness: 0.0   ← 非金属（金属感ゼロ）
roughness: 0.8 〜 1.0  ← 表面が粗くマット（光を拡散反射）

material="color: #1a1a1a; metalness: 0.0; roughness: 0.9"

予測の理由: ゴムは光をほぼ拡散反射する素材で、鏡面反射（ハイライト）がほとんど見えません。metalness を 0 に近づけると非金属の挙動になり、roughness を 1.0 に近づけると反射がぼやけてマット感が出ます。逆に roughness: 0.0 ではツルツル光るプラスチック調になり、ゴムらしさが失われます。「金属か？」「ツルツルか？」を独立に判断するのがコツです。

例題 例題2 やってみよう 解答（material.html: opacity 0.4 と 0.9 の比較）

material.html の E（半透明の赤い箱）で opacity を 0.4 → 0.9 に変更したときの観察結果:

opacity: 0.4 → 後ろのオブジェクト・床がはっきり透けて見える（赤いガラス越しのような表現）
opacity: 0.9 → ほぼ不透明な赤色。後ろのものは輪郭がうっすら分かる程度
opacity: 1.0 → 完全に不透明（transparent: true を書いていても通常の表示と同じ）

教育的要点: opacity は 0.0（完全に透明）から 1.0（完全に不透明）までを線形に変化させます。transparent: true を付けないと描画パイプラインが透明扱いをしないため、opacity を変えても見た目が変わらないことがあります。半透明オブジェクトは描画順序（重なりの前後関係）の影響も受けるため、opacity: 0.5 付近では「後ろの物が透ける／後ろのオブジェクトが消える」現象が起きることもあります。

例題 例題3 やってみよう 解答（texture.html: レンガの壁に roughness: 1.0 を追加）

texture.html のレンガの壁に roughness: 1.0 を追加したときの観察結果:

追加前: material="src: #brick-tex"
        → デフォルト roughness=0.5。レンガ表面にうっすら反射が乗り、少しテカリのある質感

追加後: material="src: #brick-tex; roughness: 1.0"
        → 完全マット。反射がなくなり、レンガが「乾いた・粗い」質感になる

material="src: #brick-tex; roughness: 1.0; metalness: 0.0"
        → さらにくっきりとマットなレンガ壁（本物のレンガに近い見た目）

教育的要点: テクスチャ画像で「色のパターン」を貼り、roughness / metalness で「質感（反射の仕方）」を制御する、という役割の分離がポイントです。同じ brick.jpg でも、roughness: 0.0 にすればツルッとした「タイルのレンガ柄」に見え、roughness: 1.0 にすれば「本物のレンガ壁」に見えます。テクスチャ ＋ マテリアルプロパティの組み合わせが3D表現のリアリティの鍵です。
