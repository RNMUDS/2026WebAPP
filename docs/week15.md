# 第15回: 成果発表・まとめ

## 今回のゴール

- 制作した作品を発表し、技術的なポイントを説明できる
- 他の受講者の作品を評価・フィードバックできる
- 授業全体で学んだWebプログラミング技術を振り返る

---

## 発表ガイド

### 発表の流れ（1人あたり5分）

| 時間 | 内容 |
|------|------|
| 1分 | 作品の紹介（コンセプト、対象ユーザ） |
| 2分 | デモンストレーション（実際に動かす） |
| 1分 | 技術的なポイント（工夫した点、苦労した点） |
| 1分 | 質疑応答 |

### 発表で話すこと

1. **作品名とコンセプト**
   - 何を作ったのか（一言で）
   - なぜこれを作ろうと思ったか

2. **デモ**
   - サーバを起動して実際に見せる
   - 複数タブでマルチユーザ機能を実演
   - 主要機能を1つずつ紹介

3. **技術的なポイント**
   - 使った技術（Express, A-Frame, Socket.IO, SQLite）
   - 特に工夫した部分（具体的に）
   - 苦労した点とどう解決したか

4. **今後の展望**
   - 時間があれば追加したかった機能
   - 改善したい点

### 発表の準備

- [ ] サーバが問題なく起動することを確認
- [ ] デモの手順を整理（何をどの順に見せるか）
- [ ] 発表メモを用意（箇条書きでOK）

## 授業全体の振り返り

### 学んだ技術マップ

```
第1-3回: サーバサイド基礎
├─ Node.js / Express.js
├─ HTTPリクエスト / レスポンス
├─ ルーティング
├─ fetch API / JSON
└─ REST API

第4-8回: 3D Web表現
├─ A-Frame
├─ エンティティ / コンポーネント
├─ ジオメトリ / マテリアル
├─ ライティング / カメラ
├─ アニメーション
└─ glTFモデル

第9-12回: リアルタイムWeb
├─ WebSocket / Socket.IO
├─ イベント駆動（emit / on）
├─ アバタ同期
├─ インタラクション同期
└─ SQLite永続化

第13-15回: 総合制作
├─ 設計（要件定義 / データ設計）
├─ 実装（デバッグ / 段階的開発）
└─ 発表（プレゼンテーション）
```

## 最終提出物

### 提出するもの

1. **プロジェクト一式**（ZIP圧縮）
   - server.js, db.js, package.json
   - public/ フォルダ一式
   - ※ `node_modules/` と `app.db` は含めない

2. **設計書**（第13回で作成したもの、最終版）

3. **スクリーンショット**（3枚以上）
   - 3D空間の全体像
   - マルチユーザ動作の様子
   - 主要機能のデモ
